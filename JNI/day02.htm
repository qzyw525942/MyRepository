第一集   JNI开发环境配置

 

交叉编译：就是在一个平台上生成另一个平台上的可执行代码。

 

NDK（Native Development Kit）就是谷歌提供的交叉编译工具。其用于Window和Linux(Android)之间的交叉编译。下载并解压缩ndk的压缩包，可以看到有个ndk-build.cmd文件，这个文件就是负责把c语言文件变成android工程中可用代码的文件，把它配置到path路径中去并在cmd中出现以下情况，说明配置成功：

除了ndk之外，还需要一个叫做CDT的插件。它在eclipse中负责c代码的高亮和提示，一般的eclipse都有这个插件了，所以无需再去安装它。

 

以上两个软件相当于java开发的jdk和ADT。

第三集   JNI开发中常见的错误

A、    java.lang.UnsatisfiedLinkError: Native method not found：有以下两种原因：

(1)如果java中native声明的方法名本身就含有下划线，而c中又按照c函数的命名规则：Java_包名_类名_java中native声明的方法名进行命名，就会出现这个问题，比如java和c中分别为：

public native String hello_FromC();

Java_com_itheima_jnihello_MainActivity_hello_FromC(...)

就会出现这个问题。解决办法是，在c中方法名中加一个1:

Java_com_itheima_jnihello_MainActivity_hello_1FromC(...)就可以了。

 

由上可见，c函数的命名规则非常麻烦。如果java中有很多native声明的方法，一个一个去敲就麻烦且易错。为此jdk（注意不是ndk）就提供了一个命令javah专门用来把java中的native方法按照c函数的命名规则生成c函数的名字。

具体用法是：如果是jdk7，那么就在android项目的src上右键

其中javah后面跟的是native方法所在的全类名。

 

运行完成后会在src目录下生成一个.h结尾的文件：

 

打开这个文件，找到以下代码，直接复制全部就能使用：

 

(2)另一种情况是，没有在java中加载.so文件：

System.loadLibrary("hello");

也会出现这种错误。

 

B、java.lang.UnsatisfiedLinkError:  .....  findLibrary returned null

(1)这种错误是因为System.loadLibrary("hello");中加载.so文件的文件名字写错了。

(2)在项目的jni路径中没有Application.mk文件的默认情况下，开发工具编译生成的.so文件只能运行在cpu为arm的手机上；如果运行在cpu为其它平台的手机上，就会报这个错误。如果要能在cpu为其它平台的手机上正常运行，就要在jni路径中增加一个Application.mk文件，内容如下：

 

APP_ABI := armeabi  x86

//如果要支持多个平台，中间用空格隔开。注意大小写不能乱，否则会出错。这//些平台在ndk的doc目录中的APPLICATION-MK.html中能找到。这里尽量复制。

第五集   JNI中的参数传递

 

现在来考虑JNI中的参数传递问题。java传递几个参数给c语言，c语言处理好后再返回给java。

 

新建一个android项目，并新建一个类：

public class JNI

{

 

        // 在哪个类中声明的native方法，就在那个类中加载.so文件

        static

        {

                System.loadLibrary("passdata");

        }

 

        // 传递两个int类型的变量给C,让C加一下返回来

        public native int add(int x, int y);

 

        // 传递String类型的参数给C,处理一下返回来

        public native String sayHelloInC(String s);

 

        // 传递int类型的数组给C,处理一下返回来

        public native int[] arrElementsIncrease(int[] intArray);

}

 

然后MainActivity如下：

C代码如下：

#include <jni.h>

#include <stdlib.h>

 

/**

 *把java传递进来的两个int类型的参数相加，再返回给java

 */

JNIEXPORT jint JNICALL Java_com_itheima_javapassdata_JNI_add

  (JNIEnv * env, jobject clazz, jint x, jint y){

  //在jint上按住ctrl健点进去可见：typedef int  jint;说明c语言的int就是java中的int

 

        return x+y;

}

 

 

/**

 * 此方法先不用管，现在还看不懂，直接使用即可，在今天的资料中有。

 *

 * 把java传递过来的String转换成c语言的char* 类型.

 */

char* _JString2CStr(JNIEnv* env, jstring jstr) {

         char* rtn = NULL;

         jclass clsstring = (*env)->FindClass(env, "java/lang/String");

         jstring strencode = (*env)->NewStringUTF(env,"GB2312");

         jmethodID mid = (*env)->GetMethodID(env, clsstring, "getBytes", "(Ljava/lang/String;)[B");

         jbyteArray barr = (jbyteArray)(*env)->CallObjectMethod(env, jstr, mid, strencode); // String .getByte("GB2312");

         jsize alen = (*env)->GetArrayLength(env, barr);

         jbyte* ba = (*env)->GetByteArrayElements(env, barr, JNI_FALSE);

         if(alen > 0) {

                rtn = (char*)malloc(alen+1); //"\0"

                memcpy(rtn, ba, alen);

                rtn[alen]=0;

         }

         (*env)->ReleaseByteArrayElements(env, barr, ba,0);

         return rtn;

}

 

 

 

/**

 *把上一个方法返回的char*类型加以处理，并转化为java中的String，再返回给java

 */

JNIEXPORT jstring JNICALL Java_com_itheima_javapassdata_JNI_sayHelloInC

  (JNIEnv * env, jobject clazz, jstring jstr){

 

        //调用工具方法把java中的String类型转换成 C语言中的 char*

        char* cstr = _JString2CStr(env,jstr);

 

        //调用strlen获取 cstr字符串的长度

        int length = strlen(cstr);

 

        int i;

        for(i = 0;i<length;i++){

 

                *(cstr+i) += 1;

                //*(cstr)指向的是首地址。

                //将每一个字符加1。类似加密处理

 

        }

 

        return (*env)->NewStringUTF(env,cstr);

}

 

/**

 *把java传递进来的int类型的数组进行处理，再返回给java

 */

JNIEXPORT jintArray JNICALL Java_com_itheima_javapassdata_JNI_arrElementsIncrease

  (JNIEnv * env, jobject clazz, jintArray jArray){

 

        //JNI.h中：jsize (*GetArrayLength)(JNIEnv*, jarray);

        jsize length =(*env)->GetArrayLength(env,jArray);

 

        //JNI.h中：jint*  (*GetIntArrayElements)(JNIEnv*, jintArray, jboolean*);

        //第三个参数一般为NULL

        //获得int数组中的每一个int元素

        int* arrayPointer =(*env)->GetIntArrayElements(env,jArray,NULL);

 

        int i;

        for(i = 0;i<length;i++){

 

                *(arrayPointer+i) += 10;

                //每一个元素加10

 

        }

 

        return jArray;

}

        // 回调参数为String的方法

/**

 *通过反射的方式调用java中的带两个int参数的方法

 */

JNIEXPORT void JNICALL Java_com_itheima_callbackjava_JNI_callbackintmethod(

                JNIEnv * env, jobject clazz)

{

        //① 获取字节码对象

        jclass claz = (*env)->FindClass(env, "com/itheima/callbackjava/JNI");

 

        //②获取Method对象

        jmethodID methodID = (*env)->GetMethodID(env, claz, "add", "(II)I");

        //(II)I：(II)表示add的方法参数为两个int类型的数据，括号外的I表示add方法的返回值为int类型

 

        //③通过字节码对象创建一个Object

        //在这里其实就是jobject clazz

 

        //④通过对象调用方法

        //jni.h中：jint (*CallIntMethod)(JNIEnv*, jobject, jmethodID, ...);

        int result = (*env)->CallIntMethod(env, clazz, methodID, 3, 4);

        //3和4是要传递给java中的add方法的参数。返回值是二者之和。注意这里没有把返回值返回给java，而是直接通过Log打印出來

 

        LOGD("result = %d", result);

}

/**

* 把java传递过来的String转换成c语言的char* 类型.

 */

char* _JString2CStr(JNIEnv* env, jstring jstr) {

 

         char* rtn = NULL;

// NULL：没有意义

 

         jclass clsstring = (*env)->FindClass(env, "java/lang/String");

        //找到String的字节码对象

 

         jstring strencode = (*env)->NewStringUTF(env,"GB2312");

         //把char*类型的GB2312转化为jstring类型

 

jmethodID mid = (*env)->GetMethodID(env, clsstring, "getBytes", "(Ljava/lang/String;)[B");

//获得String中的参数为String，返回值为byte数组([B之义)的getBytes方法

 

jbyteArray barr = (jbyteArray)(*env)->CallObjectMethod(env, jstr, mid, strencode);

//相当于：String .getByte("GB2312");

       

 jsize alen = (*env)->GetArrayLength(env, barr);

         //获得barr的长度       

 

         jbyte* ba = (*env)->GetByteArrayElements(env, barr, JNI_FALSE);

         //拿到字节数组元素的首地址

         

         if(alen > 0) {

 

                rtn = (char*)malloc(alen+1);

 //申请内存地址，加1用于存放："\0"

 

                memcpy(rtn, ba, alen);

                //数组拷贝。把字节数组拷贝到rtn申请的内存地址中   

 

                rtn[alen]=0;

                //把"\0"放入数组

         }

 

         (*env)->ReleaseByteArrayElements(env, barr, ba,0);

//释放字节数组元素

 

         return rtn;

}
