第一集    Servlet引入案例

servlet(server applet:服务器程序)属于服务器上的动态资源，它是可以接受请求并完成响应的java程序。

 

第一步：

在web工程中新建类：

public class AServlet implements Servlet

// servlet的第一种创建方式：实现Servlet接口。实现后自动添加以下方法。

         {

                   System.out.println("servlet初始化方法");

         }

         {

                   System.out.println("servlet接受请求并响应");

         // 先学上面的方法----------------------------------------------------

 

         public ServletConfig getServletConfig()

                   return null;

         }

                  return null;

         }

 

}

第二集    Servlet的方法之生命周期

 

由上面的代码可见，第一次访问servlet时，控制台打印出了”servlet初始化方法”和”servlet接受请求并响应”，说明运行了init方法和service方法；再访问几次servlet，都只打印了”servlet接受请求并响应”，说明都只执行了service方法；关闭服务器，打印出”servlet销毁方法”，说明执行了destroy方法。init方法，service方法和destroy方法构成了servlet的生命周期。

 

 

第三集    Servlet的方法之servletConfig

 

由第一集的引入案例可见，除了servlet的生命周期的三个方法之外，还有两个方法。下面来学习它们。

 

public class BServlet implements Servlet//和第一集的引入案例在同一个web工程中

{

 

         public String getServletInfo()

         {

                   // 返回servlet的作者版权等信息，无用不学。

                   return null;

         }

 

         private ServletConfig config;

 

         public ServletConfig getServletConfig()

         {// 返回 ServletConfig 对象，它包含此 servlet 的初始化和启动参数。其中初始化和启动参数是在web.xml文件中自己预先配置好的，见后面。

 

                   return config;

//返回的ServletConfig对象和传递给init方法的ServletConfig对象是同一个对象,        //所以这里返回给外面定义的private ServletConfig config;再从init方法中获得它

 

         }

 

         public void init(ServletConfig config) throws ServletException

         {//把getServletConfig返回的包含初始化和启动参数的ServletConfig传递进来用以对          //servlet进行初始化

                   this.config = config;//获得getServletConfig返回的ServletConfig。这个ServletConfig             //参数传递的步骤都是这样写的，以后都这样写就行。

         }

 

         public void destroy()

         {

 

         }

 

         public void service(ServletRequest req, ServletResponse res)

                            throws ServletException, IOException

         {

//下面是ServletConfig的几个方法

 

                   // 1、获得web.xml中配置的初始化参数的所有键,返回枚举类型

                   Enumeration<String> en = getServletConfig().getInitParameterNames();

 

                   // 2、遍历所有的键

                   while (en.hasMoreElements())

                   {

                            String key = en.nextElement();

 

                            // 3、根据键获得值

                            String value = getServletConfig().getInitParameter(key);

 

                            // 4、将键值对输出到浏览器

                            res.getWriter().write(key + "----" + value + "\n");

                            // 可见将web.xml配置的初始化参数的键值对输出到了浏览器

                   }

 

                   // 获得servlet的名称:即<servlet-name>标签的值

                   String name = getServletConfig().getServletName();

                   res.getWriter().write(name + "\n");

 

//servletConfig还有一个getServletContext()方法，后面学。

         }

}

细节一：

由servlet的生命周期方法可见，servlet只会在第一次启动时创建，也就是说一个servlet只能有一个servlet对象。这里就会出现线程安全问题：当多个人同时访问这个servelt时，因为只有一个servlet对象，后访问者会把前访问者的这个servlet对象的成员变量的值替换掉，造成混乱。如果把这些成员变量定义在成员方法中，变为局部变量，则每个用户来访问时都会执行这些成员方法，也就对应着创建它们各自的局部变量，不会造成混乱。所以以后在servlet中都把变量定义在方法中去，成为局部变量，避免线程安全问题。

第五集   ServletContext

ServletContext是属于工程级别的对象，一个工程有且只有一个ServletContext对象,它就像是工程的法人代表一样。其随着工程的启动而启动，随着工程的关闭而销毁。获得它的方式如下：

servletConfig.getServletContext();

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

                            String value = context.getInitParameter(key);

 

                            // 4、输出到浏览器

                     response.getWriter().write(key + "-----" + value + "\n");

 

                   }

 

         }

}

         }

HServlet如下：

public class HServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

                   // 1、获得servletConfig对象

                   ServletContext context = getServletContext();

 

                   // 2、从Map(域)中获得通信信息

                   Object name = context.getAttribute("name");

                   Object age = context.getAttribute("age");

 

                   response.getWriter().write(name + "---" + age);

 

                   // 3、删除键为age的键值对

                   context.removeAttribute("age");

 

                   // 4、遍历Map(域)中存放的键值对

                   Enumeration<String> names = context.getAttributeNames();

 

                   while (names.hasMoreElements())

                   {

                            String key = (String) names.nextElement();

 

                            Object value = context.getAttribute(key);

 

                            response.getWriter().write(key + "----" + value + "\n");

                            // 这里要说明的是，除了自己安排存放在Map中的键值对外，服务器也会往                     //里面存放键值对

                            // 所以打印出很多的键值对。

                   }

 

         }

 

}

C、ServletContext的第三个功能：获得工程内的资源

 

public class JServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

 

                   // 获得工程内的user.xml文件

 

                   // 1、获得servletContext

                   ServletContext context = getServletContext();

 

                   // 2、获得user.xml的路径：此时user.xml在WebRooT根目录下

 

                   // 获得相对路径

                   InputStream is = context.getResourceAsStream("/user.xml");

                   // 斜杠表示WebRoot路径

                   System.out.println(is);

 

                  // 获得绝对路径(硬盘上的路径)

                   String path = context.getRealPath("/user.xml");

                   System.out.println(path);

 

                   // 3、获得工程中不同位置的资源文件

 

                   // A获得WEB-INF下的user.xml

                   InputStream is1 = context.getResourceAsStream("/WEB-INF/user.xml");

                   System.out.println(is1);

 

                   // B获得lib下的user.xml

                   InputStream is2 = context.getResourceAsStream("/WEB-INF/lib/user.xml");

                   System.out.println(is2);

 

                   // C获得src下的user.xml：src下的java文件会被编译为class文件放在classes中

                   // 非java文件直接复制到classes中，所以此时相当于user.xml文件在classes中。

                   InputStream is3 = context

                                     .getResourceAsStream("/WEB-INF/classes/user.xml");

                   System.out.println(is3);

 

         }

 

}
