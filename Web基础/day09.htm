第二集     会话技术之cookie

 

会话技术：

就是记录服务器和浏览器之间通信信息（好比两个人的聊天记录）的技术。

 

会话的范围：

类比两个人从打招呼到互相道别是一次会话，服务器和浏览器之间从浏览器开始访问到浏览器关闭也是一次会话。

 

cookie是属于浏览器端的聊天记录；session是属于服务器端的聊天记录。

 

cookie原理如下：

 

演示：

public class CServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

 

                   Cookie c = new Cookie("name", "jerry");

 

                   // 设置cookie的有效时间

                   c.setMaxAge(60 * 60 * 24 * 7 * 2);// 告诉浏览器保存2周。此处单位为秒。

 

                   // c.setMaxAge(-1);// -1代表 在会话结束时删除cookie(默认情况)

 

                   // c.setMaxAge(0);// 通常用于删除已经存在的cookie.

 

                   response.addCookie(c);

 

         }

 

}

cookie的发送：

浏览器在什么情况下才会将cookie发送给服务器？

如上图所示，域相当于主机，域和路径一起构成了cookie所在的路径。只有浏览器访问的主机是localhost，并且路径是上图路径的子路径时，浏览器才会将cookie发送给服务器。

比如如果主机(域)是:localhost，路径是"/day10-cookie"时：

http:/localhost/day10-cookie/BServlet                               会发送

http:/localhost/day10-cookie/ABC/BCD/CServlet           会发送

http:/localhost/day12/AServlet                                            不会发送

http:/www.baidu.com/day10-cookie/BServlet                 不会发送

案例一：商品浏览历史记录

 

//获得cookie的工具类

public class CookieUtils

{

         public static Cookie getCookieByName(HttpServletRequest request, String name)

         {

                   Cookie cookie = null;

 

                   Cookie[] cookies = request.getCookies();

 

                   if (cookies != null)

                   {

                            for (Cookie c : cookies)

                            {

                                     if (name.equals(c.getName()))

                                     {

                                               cookie = c;

                                     }

                            }

                   }

 

                   return cookie;

         }

}

 

//后台servlet：

public class EServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

 

                   // 1.获得参数

                   String name = request.getParameter("name");

 

                   // 2.获得Cookie

                   Cookie history = CookieUtils.getCookieByName(request, "history");

 

                   if (history != null)

                   {

                            // 存在 => 在原来的cookie中加上当前浏览的商品，封装成如下形式：dell,hp,lenvo

                            if (!history.getValue().contains(name))

                            // 如果两次点击同一种商品，则第二次不再添加

                            {

                                     history = new Cookie("history", history.getValue() + "," + name);

                            }

                   } else

                   {

                            // 不存在 => 创建cookie

                            history = new Cookie("history", name);

                   }

 

                   // 3.将cookie发送回浏览器

                   response.addCookie(history);

 

                   // 4.重定向到JSP列表页面：list.jsp

                   response.sendRedirect("/day09-cookie/history/list.jsp");

         }

 

}

案例二：记住用户名

 

//servlet：

public class FServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

                   // 1.获得用户名密码(为简单起见，此处只需要获得用户名即可)

                   String name = request.getParameter("name");

 

                   // 2.校验用户名密码：检验失败的情况如下处理：

                   if (name == null || "".equals(name.trim()))

                   {

                            // 失败=> 转发到表单页面,并提示错误

 

                            request.setAttribute("error", "请输入用户名!");

 

                            request.getRequestDispatcher("/remember/login.jsp").forward(

                                               request, response);

 

                            return;

                   }

 

                   // 校验成功如下几步处理：

 

                   // 3.创建cookie 添加要保存的用户名,

                   Cookie c = new Cookie("remember", name);

 

                   // 4.查看记住用户名复选框是否被选中：jsp中的复选框传来一个remember参数，          //表示其是否被选中

                   String remember = request.getParameter("remember");

                   if ("yes".equals(remember))

                   {

                            // 选中 => 设置保存时间为2周

                            c.setMaxAge(60 * 60 * 24 * 7 * 2);

                   } else

                   {

                            // //没选中=>设置保存事件为0

                            c.setMaxAge(0);

                   }

 

                   // 5.添加到响应中

                   response.addCookie(c);

 

                   // 6.重定向到成功页面

                   response.sendRedirect("/day09-cookie/index.jsp");

 

         }

 

}

session的范围：

         1>浏览器第一次访问服务器,服务器创建session对象.会话开始.

         2>*浏览器关闭,保存sessionID的cookie丢失. 会话结束.(session还在服务器中)

          *session默认半个小时后过期(session不存在)

          *手动销毁session(session不存在)

public class AServlet extends HttpServlet

{

 

         public void doGet(HttpServletRequest request, HttpServletResponse response)

                            throws ServletException, IOException

         {

 

                   // 0设置正文类型

                   response.setContentType("image/jpeg");

 

                   // 1 生成验证码

                   ValidateCode code = new ValidateCode(150, 70, 4, 2);

                   // 将今天的resource路径下的ValidateCode.jar复制到本项目下的WebRoot/WEB-INF/lib路径下

                   // 因为是web项目，所以无需add to build path（java项目才需要add to build path）。

                   // 这个包就是专门用来生成验证码的工具类。

                   // 前两个参数表示验证码区域的长和宽，第三个参数表示验证码的个数，第四个参数表示干扰线的条数

 

                   // 2 将生成的正确验证码放入Session中

                   String str = code.getCode();

 

                   request.getSession().setAttribute("code", str);

 

                   System.out.println(str);

 

                   // 3 将图片流发送给浏览器

                   code.write(response.getOutputStream());

         }

 

}
