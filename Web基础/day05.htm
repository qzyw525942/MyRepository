第四集     ResultSet

 

1、获得ResultSet并遍历

 

//ResultSet细节

//功能: 封装select返回的查询结果

 

//操作: 如何获得查询结果

//结论: 

//1. next方法,向下移动并判断是否有内容

//2. getXXX方法,根据列索引或列名获得列的内容

 

public class Demo

{

         @Test

         public void fun1() throws Exception

         {

                   // 1 注册驱动

                   Class.forName("com.mysql.jdbc.Driver");

 

                   // 2 获得连接

                   Connection conn = DriverManager.getConnection(

                                     "jdbc:mysql://localhost:3306/day05", "root", "1234");

 

                   // 3 创建Statement

                   Statement st = conn.createStatement();

 

                   // 4 书写sql

                   String sql = "select * from t_user";

 

                   // 5 执行sql

                   ResultSet rs = st.executeQuery(sql);// 返回查询结果

 

                   // 向下移动一行,并判断这一行有没有数据。

                   // next返回的是boolean类型，true表示这一行有数据,反之没有

                   while (rs.next())// 如果有数据就执行循环

                   {

 

                            // 取数据:getXXX

                            int id = rs.getInt(1);

                            // 根据列索引进行取值。 获得第一列的值。JSBC涉及到的索引都是从1开始

 

                            // int id = rs.getInt("id");

                            // 根据列名称进行取值。 获得id列的值。不建议使用。

 

                            String name = rs.getString(2);// 获得第二列的值

 

                            int age = rs.getInt(3);// 获得第三列的值

 

                            System.out.println(id + "==>" + name + "==>" + age);

 

                   }

 

                   // 6关闭资源

                   st.close();

                   conn.close();

         }

        

}

 

由上面遍历的代码可见，获得每一列的数据需要getXXX方法，XXX代表每一列的类型，MySQL中的数据类型和java中的数据类型对应如下：

/*

          * 数据库类型      java类型

          * int              int

          * double          double

          * decimal         double

 

          * char            String

          * varchar         String

 

          * datetime        Date

          * timestamp       Date

          */

比如说，当表中某一列为Varchar类型时，需要使用getString方法来获取其数据。上面的getXXX方法可以统一使用getObject()方法来代替。

 

第五集     JDBC工具类

 

由以上可见，JDBC的代码有很多相同的地方，这些相同的地方可以封装在一个类中。以后需要时直接调用这个类即可。

 

首先新建一个配置文件：

 

public class JDBCUtils

{

         private static String driver;

         private static String url;

         private static String user;

         private static String password;

 

         static

         {

 

                   try

                   {

                            // 1.3 读取配置文件

                            Properties prop = new Properties();

 

                            InputStream is = new FileInputStream("src/第五集/db.properties");

 

                            prop.load(is);

 

                            is.close();

 

                            driver = prop.getProperty("driver");

                            url = prop.getProperty("url");

                            user = prop.getProperty("user");

                            password = prop.getProperty("password");

 

                            // 1.1 注册驱动：把注册驱动的部分放在静态代码块中，只运行一次，且随着                    //类的加载就运行

                            Class.forName(driver);

                   } catch (Exception e)

                   {

                            e.printStackTrace();

                   }

 

         }

 

         // 1 获得连接

         public static Connection getConnection()

         {

                   Connection conn = null;

                   try

                   {

                            // 1.2 url, user和password会随着数据库的变化而变化，所以把他们提取到配                      //置文件中去

                            // 配置文件的内容只需要读取一次就行，所以把相关的代码也放在静态代码                      //块中去。

                            conn = DriverManager.getConnection(url, user, password);

 

                   } catch (Exception e)

                   {

                            e.printStackTrace();

                   }

 

                   return conn;

         }

 

         // 2 释放资源

         // 1> 参数可能为空：需要加if判断

         // 2> 调用close方法要抛出异常,确保即使出现异常也能继续关闭：需要加tryCatch

         // 3>关闭顺序,需要从小到大

         public static void close(Connection conn, Statement st, ResultSet rs)

         {

 

                   try

                   {

                            if (rs != null)

                            {

                                     rs.close();

                            }

                   } catch (SQLException e)

                   {

                            e.printStackTrace();

                   } finally

                   {

                            try

                            {

                                     if (st != null)

                                     {

                                               st.close();

                                     }

                            } catch (SQLException e)

                            {

                                     e.printStackTrace();

                            } finally

                            {

                                     try

                                     {

                                               if (conn != null)

                                               {

                                                        conn.close();

                                               }

                                     } catch (SQLException e)

                                     {

                                               e.printStackTrace();

                                     }

                            }

 

                   }

 

         }

 

 

第六集     SQL注入

 

1、SQL注入问题

 

2、使用prepareStatement就能解决这个问题。prepareStatement分两次运送SQL给数据库，第一次运送不带参数的SQL语句，第二次再把参数运送给数据库，把SQL和参数分开运送就不存在SQL注入问题了。

 

案例：

 

public class Demo

{

//使用PrepareStatement对象,不存在sql注入问题

         @Test

         public void fun2() throws Exception

         {

                   String name = "xxx' OR 1=1 -- ";

                   String password = "1234";

 

                   // 1 获得连接

                   Connection conn = JDBCUtils.getConnection();

 

                   // 2 拼装sql语句

                   String sql = "SELECT * FROM t_user  WHERE NAME=? AND   PASSWORD=?";

                   // 参数部分用？代替

 

                   // 3 获得PrepareStatement

                   PreparedStatement ps = conn.prepareStatement(sql);

                   // 刚创建就需要sql，先把sql运送给数据库

 

                   // 4 设置参数到ps对象中

                   ps.setString(1, name);

                   ps.setString(2, password);

                   // name和password在数据库中都是varchar，所以这里用String

                   // 1代表第一个问号，2代表第二个问号

 

                   // 5 运送参数,执行sql并拿到结果

                   ResultSet rs = ps.executeQuery();

                   // 与statement的一样，增删改用executeUpdate,查询用executeQuery

 

                   // 5 根据结果判断是否登录成功

                   if (rs.next())

                   {

                            System.out.println("登录成功!");

                   } else

                   {

                            System.out.println("登录失败!");

                   }

 

                   // 6关闭资源

                   JDBCUtils.close(conn, ps, rs);

         }

}

案例1：向mysql中存放大文本文件

 

准备工作：先在数据库中创建一张表：

create table mytext(

                   id int primary key AUTO_INCREMENT,

                   text text

)

并把一个文本文件text.txt放在工程目录下。

 

public class Demo {

 

         @Test

         //存储大文本必须使用PrepareStatement对象

         public void fun1() throws Exception{

                  

                   //1 获得连接

                   Connection conn = JDBCUtils.getConnection();

 

                   //2 书写sql

                   String sql = "insert into mytext values(null,?)";//id是自增的，给予null其会自增

 

                   //3 创建PrepareStatement

                   PreparedStatement ps = conn.prepareStatement(sql);

 

                   //4 设置参数

                  

                   File f = new File("src/text.txt");

                  

                   FileReader reader = new FileReader(f);

                  

                   ps.setCharacterStream(1, reader, (int)f.length());

//text类型的要使用setCharacterStream方法。

//参数1:参数的索引：第一个问号

                   //参数2:需要保存的文本的流

                   //参数3:文件长度。注意默认为long类型的，MySQL不支持，所以要转为int                       //类型

 

                  

                   //5 执行sql

                   int result = ps.executeUpdate();//插入语句用Updata方法

                   System.out.println(result);

 

                   //6关闭资源

                   JDBCUtils.close(conn, ps, null);

                   }

}

 

案例2：批量执行sql语句;

 

public class Demo

{

 

// 1 使用Statement对象批量执行sql

         @Test

         public void fun1() throws Exception

         {

 

                   // 1 获得连接

                   Connection conn = JDBCUtils.getConnection();

 

                   // 2 获得Statement

                   Statement st = conn.createStatement();

 

                   // 3 添加多条sql语句到st中

                   st.addBatch("create table t_stu ( id int primary key auto_increment ,

 name varchar(20) )");

                   st.addBatch("insert into t_stu values(null,'tom')");

                   st.addBatch("insert into t_stu values(null,'jerry')");

                   st.addBatch("insert into t_stu values(null,'jack')");

                   st.addBatch("insert into t_stu values(null,'rose')");

 

                   // 4 执行sql

                   int[] results = st.executeBatch();// 批量sql不支持查询

                   System.out.println(Arrays.toString(results));

 

                  // 5关闭资源

                   JDBCUtils.close(conn, st, null);

         }

 

 

// 2 使用PrepareStatement对象批量执行sql

         @Test

         public void fun2() throws Exception

         {

 

                   // 1 获得连接

                   Connection conn = JDBCUtils.getConnection();

 

                   // 2 书写sql语句

                   String sql = "insert into t_stu values(null,?)";

 

                   // 3 创建PrepareStatement

                   PreparedStatement ps = conn.prepareStatement(sql);

 

                   // 4 循环.添加参数

                   for (int i = 0; i < 100; i++)

                   {

                            ps.setString(1, "用户" + i);

                            ps.addBatch();// 表示参数设置完毕，保存起来。共执行100次

                   }

 

                   // 5 批量执行

                   int[] results = ps.executeBatch();

                   System.out.println(Arrays.toString(results));

 

                   // 5关闭资源

                   JDBCUtils.close(conn, ps, null);

         }

}
