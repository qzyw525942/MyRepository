第一集  xml解析

 

xml解析：通过程序把xml文件中的数据读取出来。

 

A.DOM解析：

把文档中所有内容都拆解并封装成不同的对象.通过这些对象来操作文本的内容。

 

                   对象种类:

                            Document<==>整个文档

                            Element<==>标签

                            Attribute<==>属性

                            Text<==>标签内的文本

                            Common<==>注释

 

Dom解析的优势:频繁增删改查时很方便.

Dom解析的劣势:解析整篇文档，资源消耗比较大,不适于安卓。

 

B、SAX解析：

使用如下这些事件去解析xml文档。

        

         文档开始事件: startDocument

         文档结束事件: endDocument

         元素开始事件: startElement

         元素结束事件: endElement

         文本事件:character

 

优点: 只解析所需要的xml内容，效率高，节省资源。

缺点: 没有得到整篇文档. 无法进行增删改的操作，只能查询。

 

C、pull解析：

安卓使用此方式。

        

         sax一旦开始解析就要把预设的部分文档从头读到尾.不能中途停止。

         pull与SAX差不多，手动向下继续解析. 如果获得到要找的内容. 可以停止继续解析，    即可以中途停止。

解析类如下：

public class PullTool

{

         // 使用Pull解析

         public static List<Student> parserXml(InputStream is) throws Exception

         {

                   // 把xml文件中的内容读取出来，首先要获得xml文件，所以参数设置为字节输              //入流； xml文件中的内容是多个Student，读取出来后应该放在Student类的对              //象中，这些对象应该放在集合中，所以返回值为 List<Student>

 

                   // 0

                   List<Student> list = null;

                   Student stu = null;

 

                   // 1 创建解析器工厂

                   XmlPullParserFactory factory = XmlPullParserFactory.newInstance();

 

                   // 2 使用工厂获得解析器

                   XmlPullParser parser = factory.newPullParser();

 

                   // 3 使用解析器读取xml流

                   parser.setInput(is, "UTF-8");

 

//开始解析xml文件时指针位于第一行文档声明处，然后一行一行的往下读取。pull         //解析需掌握四个事件:

START_DOCUMENT，END_DOCUMENT，START_TAG，END_TAG.

 

 

                   // 4 获得当前指针的位置对应的状态码

                   int type = parser.getEventType();//获得当前事件类型属于上面四个类型中的哪一个。

 

                   // 判断当前事件是否是文档结束事件

                   while (type != XmlPullParser.END_DOCUMENT)

                   {

                            switch (type)

                            {

 

                            case XmlPullParser.START_TAG:

 

                                     //元素开始标签有很多个，所以要判断

//判断当前遇到的元素名称是否为students

                                     if ("students".equals(parser.getName()))

                                     {

//parser.getName：通过解析器获得当前遇到的元素名称是什么。

 

                                               list = new ArrayList<Student>();//创建集合对象

 

                                     } else if ("student".equals(parser.getName()))

                                     {

                                               // 初始化Student对象

                                               stu = new Student();

 

                                               // 读取number属性

                                               String number = parser.getAttributeValue(null, "number");

                                               stu.setNumber(number);

 

                                     } else if ("name".equals(parser.getName()))

                                     {

                                               String name = parser.nextText();

                                               stu.setName(name);

 

                                     } else if ("sex".equals(parser.getName()))

                                     {

                                               String sex = parser.nextText();

                                               stu.setSex(sex);

 

                                     } else if ("age".equals(parser.getName()))

                                     {

                                               int age = Integer.parseInt(parser.nextText());

                                               stu.setAge(age);

                                     }

 

                                     break;

 

                            case XmlPullParser.END_TAG:

 

                                     if ("student".equals(parser.getName()))

                                     {

                                               list.add(stu);

                                               stu = null;

                                     }

                                     break;

                            default:

                                     break;

                            }

 

                            // 指针跳到下一行,并返回该行的事件类型属于四种事件中的哪一种。

//这一行不可忘

                            type = parser.next();

                   }

                   return list;

         }

 

 

         public static void main(String[] args) throws Exception

         {

                   FileInputStream is = new FileInputStream("src/stu.xml");

 

                   List<Student> list = PullTool.parserXml(is);

 

                   System.out.println(list);

//通过Student中的toString方法把Student中的内容打印出来

 

                   is.close();

 

         }

}

第三集    TomCat服务器

 

装有服务器软件的计算机叫做服务器，比如机房的柜式服务器，其本质也是计算机，不过被改变了以更适应服务工作，里面装有服务器软件。

 

TomCat服务器指的是服务器软件。

 

解压完TomCat的压缩包后，在解压出来的文件中找到bin/startup.bat,双击即可启动TomCat服务器。此时会出来一个cmd窗口，它是服务器的运行日志窗口，不能关闭。

 

在浏览器中输入localhost:8080，回车后如果能访问到TomCat主页，说明TomCat安装成功。

 

在弹出的cmd窗口中按ctrl+c可关闭TomCat。

 

 

TomCat的默认端口号为8080，其值可以修改。在上图的conf目录中，使用Notepad++打开server.xml，在69行有如图所示代码：

修改为80后，重启服务器，此后就可以直接通过localhost访问服务器了。

 

浏览器访问服务器的过程如下：

 

静态web项目：

项目中全部资源都是静态资源。

动态web项目：

只要项目中有动态资源，则该项目就是动态web项目。

 

在Tomcat中部署静态项目：

在webapps中新建hello文件夹，在文件夹中新建文件hello.html文件；启动服务器；在浏览器中通过localhost/hello/hello.html就可以访问了。

 

在Tomcat中部署动态项目：

动态web项目需要满足如下目录结构

                   |-WEB-INF 文件夹=> 项目配置文件夹.浏览器不能访问其内容.

                            |-classes文件夹 => 放置web项目的字节码文件.

                            |-lib文件夹     => 项目中要使用的jar包

                            |-web.xml文件        => web项目唯一配置文件
