IP地址:每台计算机的“身份证号”

        

                   IP地址：192.168.1.100

                  

         IP地址的组成：网络号段+主机号段

                   A类IP地址：第一号段为网络号段(192)+后三段为主机号段(168.1.100)

                   所以一个网络号可以配置256*256*256 = 16777216台主机

                   B类IP地址：前两号段为网络号段(192.168)+后两段为主机号段(1.100)

                   所以一个网络号可以配置256*256 = 65536台主机

                   C类IP地址：前三号段为网络号段(192.168.1)+后一段为主机号段(100)

                   所以一个网络号可以配置256台主机

 

         IP地址的范围：

                   A类IP地址：从1.0.0.1到127.255.255.254       

                   B类IP地址：从128.0.0.1到191.255.255.254

                   C类IP地址：从192.0.0.1到223.255.255.254  

                  

常用的是C类IP地址

 

         两个DOS命令：

                   Ipconfig：查看本机ip地址

      

                   Ping：后面跟ip地址。测试本机与指定的ip地址间的通信是否有问题

                  

         特殊的IP地址：

                   127.0.0.1 回环地址(也表示本机)

                   x.x.x.255 广播地址

                   x.x.x.0 网络地址

协议：通信的规则

                                                                

         UDP协议：

数据大小有限制

                   需要把数据打包

                   不建立连接，所以速度快，不可靠

                   数据大小无限制

                   建立连接通道，所以速度慢，可靠

public class SendDemo

{

         public static void main(String[] args) throws Exception

         {

                   // 创建发送端Socket对象，UDP的Socket用的是DatagramSocket，gram：                          //义为包

                   // DatagramSocket()

                   DatagramSocket ds = new DatagramSocket();

 

                   // 创建数据，并把数据打包

                   // DatagramPacket(byte[] buf, int length, InetAddress address, int port)

                   // 第一个参数是数据，第二个参数是数据长度，第三个参数是对方的Ip                      //地址，第四个参数是对方的端口号

                   // 创建数据

                   byte[] bys = "hello,udp,我来了".getBytes();

                   // 长度

                   int length = bys.length;

                   // IP地址对象

                   InetAddress address = InetAddress.getByName("192.168.12.92");

                   // 端口

                   int port = 10086;

 

                   DatagramPacket dp = new DatagramPacket(bys, length, address, port);

 

                   // 调用Socket对象的发送方法发送数据包

                   // public void send(DatagramPacket p)

                   ds.send(dp);

 

                   // 释放资源

                   ds.close();

         }

}

// 只运行此程序没问题，因为UDP不建立连接，直接就发送过去，不管对方程序   //启动没有

 

 * UDP协议接收数据：

 * A:创建接收端Socket对象

 * B:创建一个数据包(接收容器)

 * C:调用Socket对象的接收方法接收数据

 * D:解析数据包，并显示在控制台

 * E:释放资源

 

public class ReceiveDemo

{

         public static void main(String[] args) throws IOException

         {

                   // 创建接收端Socket对象

                   // DatagramSocket(int port)

                   DatagramSocket ds = new DatagramSocket(10086);

                   // 端口号要和发送端指定的端口号一样

 

                   // 创建一个数据包(接收容器)

                   // DatagramPacket(byte[] buf, int length)

                   byte[] bys = new byte[1024];

                   // 先假定发送端的数据为1024个字节，实际上要小，

                   // 相当于先给了一个500ml的瓶子，但实际要装的水没这么多

                   int length = bys.length;

                   DatagramPacket dp = new DatagramPacket(bys, length);

 

                   // 调用Socket对象的接收方法接收数据

                   // public void receive(DatagramPacket p)

                   ds.receive(dp);

                   // 阻塞式方法,即程序走到这里就一直等着，直到发送端发送了数据后此                     //程序才往后走

                   // 所以运行的时候要先运行接收的程序，它先等着了，再运行发送的程                      //序，数据就能过来了

 

                   // 解析数据包，并显示在控制台

                   // 获取发送方的ip

                   // public InetAddress getAddress()

                   InetAddress address = dp.getAddress();

                   String ip = address.getHostAddress();

                   // public byte[] getData():获取数据缓冲区

                   // public int getLength():获取数据的实际长度

                   byte[] bys2 = dp.getData();

                   int len = dp.getLength();// 实际的数据大小

                   String s = new String(bys2, 0, len);

                   System.out.println(ip + "传递的数据是:" + s);

 

                   // 释放资源

                   ds.close();

         }

}

案例二：通过多线程实现在一个窗口发送和接收数据

 

//main方法：

public class ChatRoom {

         public static void main(String[] args) throws IOException {

                   DatagramSocket dsSend = new DatagramSocket();

                   DatagramSocket dsReceive = new DatagramSocket(12306);

 

                   SendThread st = new SendThread(dsSend);

                   ReceiveThread rt = new ReceiveThread(dsReceive);

 

                   Thread t1 = new Thread(st);

                   Thread t2 = new Thread(rt);

 

                   t1.start();

                   t2.start();

         }

}

如果此时就只开启发送端，则会出现异常，连接被拒绝。

ConnectException: Connection refused: connect

因为TCP协议一定要先开接收端。然后发送端再去连接。

 

接收端

 * A:创建接收端的Socket对象

 * B:监听客户端连接。返回一个对应的Socket对象

 * C:获取输入流，读取数据显示在控制台

 * D:释放资源

 */

public class ServerDemo

{

         public static void main(String[] args) throws IOException

         {

                   // 创建接收端的Socket对象：此时使用的是ServerSocket类

                   // ServerSocket(int port)

                   ServerSocket ss = new ServerSocket(8888);

 

                   // 监听发送端连接。返回一个对应的Socket对象

                   // public Socket accept()

                   Socket s = ss.accept(); //侦听并接受发送端套接字的连接。此方法在发送端套接字连                                                 //接之前一直阻塞等待。

 

                   // 获取输入流，读取数据显示在控制台

                   InputStream is = s.getInputStream();

 

                   byte[] bys = new byte[1024];//预定义一个接收容器

                   int len = is.read(bys); // 阻塞式方法，等待发送端发送数据后再往后走

 //只读一次就能读完

                   String str = new String(bys, 0, len);

 

                   String ip = s.getInetAddress().getHostAddress();//获取发送端的IP地址

 

                   System.out.println(ip + "---" + str);

 

                   // 释放资源

                   s.close();

                   // ss.close();

                   // 服务器的Socket ss不应该关闭，它代表的是服务器，应该关闭的是服务器中每            //个客户端对应的Socket s

         }

}

案例五：改进案例四，增加反馈功能

 

发送端：

public class UploadClient {

         public static void main(String[] args) throws IOException {

                   // 创建客户端Socket对象

                   Socket s = new Socket("192.168.12.92", 11111);

 

                   // 封装文本文件

                   BufferedReader br = new BufferedReader(new FileReader(

                                     "InetAddressDemo.java"));

                   // 封装通道内流

                   BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(

                                     s.getOutputStream()));

 

                   String line = null;

                   while ((line = br.readLine()) != null) { // 阻塞

                            bw.write(line);

                            bw.newLine();

                            bw.flush();

                   }

                  

                  s.shutdownOutput();

//上面这个while循环中，如果到达文件末尾即判断为null就结束循环，但是这个    //null是不会到达接收端的，而接收端还需要这个null来进行判断以结束循环，所 //以通过这个方法来告诉接收端已经到达文件末尾，无需再等待

 

                   // 接收反馈

                   BufferedReader brClient = new BufferedReader(new InputStreamReader(

                                     s.getInputStream()));

                   String client = brClient.readLine(); // 阻塞

                   System.out.println(client);

 

                   // 释放资源

                   br.close();

                   s.close();

         }

}

 

接收端：

public class UploadServer {

         public static void main(String[] args) throws IOException {

                   // 创建服务器端的Socket对象

                   ServerSocket ss = new ServerSocket(11111);

 

                   // 监听客户端连接

                   Socket s = ss.accept();// 阻塞

 

                   // 封装通道内的流

                   BufferedReader br = new BufferedReader(new InputStreamReader(

                                     s.getInputStream()));

                   // 封装文本文件

                   BufferedWriter bw = new BufferedWriter(new FileWriter("Copy.java"));

 

                   String line = null;

                   while ((line = br.readLine()) != null) { // 阻塞。这个null表示发送端发送的文件已经                                                                                  //达到文件末尾了。但是接收端不能收到发送端的                                                                            //null标记并不知道已经到达文件末尾，所以发送                                                                                    //端要通过shutdownOutput方法来告诉接收端已                                                                               //经到达文件末尾,以免接收端一直在这儿等着，导                                                                                    //致下面的反馈信息不能反馈给发送端

                            bw.write(line);

                            bw.newLine();

                            bw.flush();

                   }

 

                   // 给出反馈

                   BufferedWriter bwServer = new BufferedWriter(new OutputStreamWriter(

                                     s.getOutputStream()));

                   bwServer.write("文件上传成功");

                   bwServer.newLine();

                   bwServer.flush();

 

                   // 释放资源

                   bw.close();

                   s.close();

         }

}

 

案例六：上传图片

public class UploadClient {

         public static void main(String[] args) throws IOException {

                   // 创建客户端Socket对象

                   Socket s = new Socket("192.168.12.92", 19191);

 

                   // 封装图片文件：字节流

                   BufferedInputStream bis = new BufferedInputStream(new FileInputStream(

                                     "林青霞.jpg"));

                   // 封装通道内的流：字节流

                   BufferedOutputStream bos = new BufferedOutputStream(s.getOutputStream());

 

                   byte[] bys = new byte[1024];

                   int len = 0;

                   while ((len = bis.read(bys)) != -1) {

                            bos.write(bys, 0, len);

                            bos.flush();//字节流也有flush方法用以刷新数据，否则图片会残缺不全

                   }

                  

                   s.shutdownOutput();

 

                   // 读取反馈

                   InputStream is = s.getInputStream();

                   byte[] bys2 = new byte[1024];

                   int len2 = is.read(bys2);

                   String client = new String(bys2, 0, len2);

                   System.out.println(client);

 

                   // 释放资源

                   bis.close();

                   s.close();

         }

}
