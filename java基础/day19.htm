                  A:JVM的默认处理

                           把异常的名称,原因,位置等信息输出在控制台，如：CalssCastException。但是                    程序不能继续执行。

try...catch...finally的处理格式：

                try {

                         可能出现问题的代码;

                }catch(异常名 变量) {

                         针对问题的处理;

                }finally {

                         释放资源;

                }

 

 变形格式：

                try {

                         可能出现问题的代码;

                }catch(异常名 变量) {

                         针对问题的处理;

                 }

 

  注意：

                A:try里面的代码越少越好

                B:catch里面必须有内容，哪怕是给出一个简单的提示

               A:编译时异常和运行时异常的区别

编译期异常：Java程序必须显示处理，否则程序就会发生错误，无法通过编译。具         体表现为编写代码的时候就要求添加tryCatch语句或者抛出Throws,否则代码就会       出错，如下图：

 

 

                  运行期异常：无需显示处理，也可以和编译时异常一样处理。具体表现为编译期间                  代码没报错，但是一旦运行就会出错，如下图，可以把出错的代码也用tryCatch语              句包围起来。

 

         B:throw和throws的区别

 

throw:如果出现了异常，可以把该异常抛出，这个时候抛出的是异常的对象。

 

         throw

                  用在方法体内，跟的是异常对象名

                  只能抛出一个异常对象名

                  表示抛出异常，由方法体内的语句处理

                  执行throw就一定抛出了某种异常

 

         throws

                  用在方法声明后面，跟的是异常类名

                  可以跟多个异常类名，用逗号隔开

                  表示抛出异常，由该方法的调用者来处理

                  throws表示出现异常的一种可能性，并不一定会发生这些异常

 

public class ExceptionDemo {

         public static void main(String[] args) {

                  // method();

                 

         try {

                           method2();//调用非RunTimeExcettion异常的方法，调用者必须处理

                  } catch (Exception e) {

                           e.printStackTrace();

                  }

         }

 

         public static void method() {

                  int a = 10;

                  int b = 0;

                  if (b == 0) {

                           throw new ArithmeticException();//throw后面跟的是异常的对象，表                                //示异常一定发生

                  } else {

                           System.out.println(a / b);

                  }

         }

 

         public static void method2() throws Exception {

                  int a = 10;

                  int b = 0;

                  if (b == 0) {

                           throw new Exception();//要求添加抛出异常声明，所以抛出了异常

                  } else {

                           System.out.println(a / b);

                  }

         }

}

 

选用原则：

如果该功能内部可以将该问题处理就用try，如果处理不了交由调用者处理就用throws。

区别：后续程序需要继续执行就用try，后续程序不需要继续执行就用throws。

 

(5)finally关键字及其面试题

 

A:finally用于释放资源，在IO流操作和数据库操作中会见到。它的代码永远会执行，但是如果在执行到finally之前jvm退出，则finally就不会执行

                                                                                                                                

public class FinallyDemo {

         public static void main(String[] args) {

                  String s = "2014-11-20";

                  SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

 

                  Date d = null;

                  try {

                           d = sdf.parse(s);

                  } catch (ParseException e) {

                           e.printStackTrace();

                           System.exit(0);//执行到finally之前程序退出了，则finally就不能被执行

                  } finally {

                           System.out.println("这里的代码是可以执行的");

                  }

 

                  System.out.println(d);

         }

}

 

B:面试题

                  a:final,finally,finalize的区别?

 

final：最终的意思，可以修饰类，成员变量，成员方法

                修饰类，类不能被继承

                修饰变量，变量是常量

                修饰方法，方法不能被重写

 

finally：是异常处理的一部分，用于释放资源。一般来说，代码肯定会执行，但是   如果在执行到finally之前jvm退出，则finally就不会执行

 

finalize：是Object类的一个方法，用于垃圾回收

 

                  b:如果在catch里面有return,请问finally还执行吗?如果执行,在return前还是return          后呢？

                 

答案是：会，前。

 

public class FinallyDemo2 {

         public static int getInt() {

                           a = 30;

在执行到return a这一步的时候，已经不是return a而是return 30了，但是程序发现后面还有finally，所以继续执行finally的内容，把40再付给a,但是由于return 30而不再是return a,所以40不能再返回。

                  } finally {

                  }

         return a;//只能执行一个return，所以这一个不能再执行

         }

需求：E:\评书\三国演义下面的视频原来为：

名称太复杂，现要求名称修改为：00?_介绍.avi

 

  思路：

                 A:封装目录

                 B:获取该目录下所有的文件的File数组

                 C:遍历该File数组，得到每一个File对象

                  D:拼接一个新的名称，然后重命名即可。

 

public class FileDemo {

         public static void main(String[] args) {

                  // 封装目录

                  File srcFolder = new File("E:\\评书\\三国演义");

 

                  // 获取该目录下所有的文件的File数组

                  File[] fileArray = srcFolder.listFiles();

 

                  // 遍历该File数组，得到每一个File对象

                  for (File file : fileArray) {

                  //原来为E:\评书\三国演义\三国演义_001_[评书网-今天很高兴,明天就IO                     //了]_桃园三结义.avi

                  // 改后：E:\评书\三国演义\001_桃园三结义.avi

                  String name = file.getName(); // 得到三国演义_001_[评书网-今天很高兴,                        //明天就IO了]_桃园三结义.avi

 

//下面截取name

                           int index = name.indexOf("_");//得到第一个下划线的索引

                           String numberString = name.substring(index + 1, index + 4);

                           // System.out.println(numberString);//打印出001等编号

 

                           int endIndex = name.lastIndexOf('_');//得到最后一个下划线的索引

                           String nameString = name.substring(endIndex);//从最后一个下划线一                                 //直截取到最后

                           String newName = numberString.concat(nameString); // 字符串拼接                                    //001_桃园三结义.avi

                           // System.out.println(newName);

 

                           File newFile = new File(srcFolder, newName); // E:\\评书\\三国演义                                      //\\001_桃园三结义.avi

 

                           // 重命名即可

                           file.renameTo(newFile);

                  }

         }

}
