第一集  集合对象的toString方法

 

代码如下：

Collection c = new ArrayList();

       c.add("hello");

       c.add("world");

       c.add("java");

      

       System.out.println(c);//默认调用的是toString方法，但是toString方法有两种情况，要                                   //么是Object类中的，要么是类自己重写的，这里属于哪一种呢？

 

如果使用的是Object类的，那么打印出来的是c的地址值，但这里打印出的是[hello，    world,java],说明是类自己重写的，现在找到这个重写的toString方法。

A:Collection c = new ArrayList();这是多态的方式，所以c调用toString方法，其实调用 的是ArrayList的toString方法。

B:但是进入ArrayList类点击outLine视图却发现没有toString方法，进一步发现，ArrayList      继承自一个父类AbstractList,进入这个父类的outLine视图发现也是没有toString方法，      但是这个父类还有个父类AbstractCollection，进入这个父类的outLine视图发现终于找 到toString方法的源码：

 

public String toString() {

        Iterator<E> it = iterator();//相当于Iterator<E> it =this.iterator();this即集合c

        if (! it.hasNext())//如果集合没有元素就返回[]

            return "[]";

 

        StringBuilder sb = new StringBuilder();

        sb.append('[');

        for (;;) {//死循环，直到全部元素追加完毕

            E e = it.next();

            sb.append(e == this ? "(this Collection)" : e);//e是元素，this是集合，所                        //以二者不等,所以就把元素e追加到sb变量

            if (! it.hasNext())

                return sb.append(']').toString();//如果全部元素追加完毕，就在最后追                                //加“]”,并调用sb的toString方法转化为String

            sb.append(',').append(' ');//如果元素还没有追加完毕就走这儿

        }

    }

B:为什么HashSet集合能保证唯一性呢？

 

       通过查看HashSet的add方法，可知这个方法底层其实是hashMap类的put方法，所       以找到hashMap类的put方法的源码（好像已经改版了），这个方法依赖两个方法：     hashCode()和equals()。

  步骤：

            首先比较哈希值（两个对象哈希值相同不一定说明两个对象是同一个对象，还要用          equals比较，但若两个对象哈希值不相同则两对像一定不是同一个对象）

            如果相同，继续走equals()

            如果不同,就直接添加到集合中  

  按照方法的步骤来说：    

            先看hashCode()值是否相同，它返回哈希值

                   相同:继续走equals()方法

                          返回true：说明元素重复，就不添加

                          返回false：说明元素不重复，就添加到集合

                   不同：就直接把元素添加到集合      

 

如果一个类没有重写hashCode()和equals()这两个方法，那么这个类的对象就默认使用Object()的这两个方法，这样的话这些对象的的哈希值不会相同，就无法进行判断，就全都会存入set集合。而String类重写了hashCode()和equals()方法，所以，它就可以进行判断，从而把内容相同的字符串去掉，只留下一个。

 

            C:如何保证元素唯一性的呢?

                     由hashCode()和equals()保证的

 

D:HashSet存储自定义对象并遍历，并保证元素的唯一性(如果两个对象的成员变量值相同即为同一个元素)

       }

 

       @Override

                     public int hashCode() {

用HashSet存储字符串的时候，因为String类重写了hashCode()和equals()方法，所以能自动去掉重复的字符串元素。而用HashSet存储自定义对象的时候，要在工具类中添hashCode方法和equals方法（alt+shift+s+h）.因为HashSet底层依赖的是hashCode()和equals()方法。如果这两个方法在学生类中没有重写，默认使用的是Object类的。这个时候，这个类的对象们的哈希值是不会一样的，不能继续判断,就会全都存入集合，不能保证元素唯一。

              final int prime = 31;

              int result = 1;

              result = prime * result + age;

              result = prime * result + ((name == null) ? 0 : name.hashCode());

              return result;

       }

 

       @Override

       public boolean equals(Object obj) {

              if (this == obj)

                     return true;

              if (obj == null)

                     return false;

              if (getClass() != obj.getClass())

                     return false;

              Student other = (Student) obj;

            B:创建一个HashSet集合

public class HashSetDemo

{

       public static void main(String[] args)

       {

              // 创建随机数对象

              Random r = new Random();

 

              // 创建一个Set集合

              HashSet<Integer> ts = new HashSet<Integer>();

              // Integer已经实现Comparable接口，所以可以自动排序及元素是否重                 //复

 

              // 判断集合的长度是不是小于10

              while (ts.size() < 10)

              {

                     int num = r.nextInt(20) + 1;

                     ts.add(num);

              }

 

              // 遍历Set集合

              for (Integer i : ts)

              {

                     System.out.println(i);

              }

       }

}

public class TreeSetDemo

       public static void main(String[] args)

       {

              // 创建一个TreeSet集合

              TreeSet<Student> ts = new TreeSet<Student>(new Comparator<Student>()

              {

                     @Override

                     public int compare(Student s1, Student s2)

                     {

                            // 总分从高到低

                            int num = s2.getSum() - s1.getSum();// s2在前s1在后才能从高到低排，                                                                                  //反之是从低到高排

                            // 总分相同的不一定语文相同

                            int num2 = num == 0 ? s1.getChinese() - s2.getChinese() : num;

                            // 总分相同的不一定数学相同

                            int num3 = num2 == 0 ? s1.getMath() - s2.getMath() : num2;

                            // 总分相同的不一定英语相同

                            int num4 = num3 == 0 ? s1.getEnglish() - s2.getEnglish() : num3;

                            // 姓名还不一定相同呢

                            int num5 = num4 == 0 ? s1.getName().compareTo(s2.getName()) : num4;

                            return num5;

                     }

              });

 

              System.out.println("学生信息录入开始");

              // 键盘录入5个学生信息

              for (int x = 1; x <= 5; x++)

                     System.out.println("请输入第" + x + "个学生的数学成绩:");

                     System.out.println("请输入第" + x + "个学生的英语成绩:");

                     String englishString = sc.nextLine();

 

                     // 把数据封装到学生对象中

              System.out.println("学习信息从高到低排序如下：");

3:Collection集合总结

       Collection

              |--List    有序(存储顺序和取出顺序),可重复

                     |--ArrayList

                            底层数据结构是数组，查询快，增删慢。

                            线程不安全，效率高

                     |--Vector

                            底层数据结构是数组，查询快，增删慢。

                            线程安全，效率低

                     |--LinkedList

                            底层数据结构是链表，查询慢，增删快。

                            线程不安全，效率高

              |--Set     无序(存储顺序和取出顺序),唯一

                     |--HashSet

                            底层数据结构是哈希表。

                            如何保证元素唯一性的呢?

                                   依赖两个方法：hashCode()和equals()

                                   开发中自动生成这两个方法即可

                            |--LinkedHashSet

                                   底层数据结构是链表和哈希表

                                   由链表保证元素有序

                                   由哈希表保证元素唯一

                     |--TreeSet

                            底层数据结构是红黑树。

                            如何保证元素排序的呢?

                                   自然排序

                                   比较器排序

                            如何保证元素唯一性的呢?

                                   根据比较的返回值是否是0来决定

                                  

4:针对Collection集合我们到底使用谁呢?

       唯一吗?

              是：Set

                     排序吗?

                            是：TreeSet

                            否：HashSet

              如果你知道是Set，但是不知道是哪个Set，就用HashSet。

                    

              否：List

                     要安全吗?

                            是：Vector

                            否：ArrayList或者LinkedList

                                   查询多：ArrayList

                                   增删多：LinkedList

              如果你知道是List，但是不知道是哪个List，就用ArrayList。

      

       如果你知道是Collection集合，但是不知道使用谁，就用ArrayList。

      

       如果你知道用集合，就用ArrayList。

      

5:在集合中常见的数据结构

       ArrayXxx:底层数据结构是数组，查询快，增删慢

       LinkedXxx:底层数据结构是链表，查询慢，增删快

       HashXxx:底层数据结构是哈希表。依赖两个方法：hashCode()和equals()

       TreeXxx:底层数据结构是二叉树。两种方式排序：自然排序和比较器排序
