//定义人的抽象类

abstract class Person {

       private String name;

       private int age;

      

       public Person() {}

      

       public Person(String name,int age) {

              this.name = name;

              this.age = age;

       }

      

       public String getName() {

              return name;

       }

      

       public void setName(String name) {

              this.name = name;

       }

      

       public int getAge() {

              return age;

       }

      

       public void setAge(int age) {

              this.age = age;

       }

      

       //睡觉

       public void sleep() {

              System.out.println("人都是要睡觉的");

       }

      

       //吃饭

       public abstract void eat();

}

 

//定义运动员抽象类

abstract class Player extends Person {

       //因为是抽象的类，所以不重写父类的抽象方法

       public Player() {}

      

       public Player(String name,int age) {

              super(name,age);

       }

      

       //学习

       public abstract void study();

}

第四集   权限修饰符

 

(1)权限修饰符

              本类      同一个包下（子类和无关类）    不同包下的子类       不同包下的无关类

private          Y

默认             Y            Y

protected     Y         Y                                                   Y

public           Y            Y                                                      Y                        Y

 

Y：是能访问的意思

 

如果要限制只能在本类中访问，则用private修饰，如果限制在同一路径下(包中)访问，则用默认，如果限制只能在子类中访问，则用protected修饰，如果限制在任何情况下都能访问，则用public修饰

 

//同一个类中全部都可以访问

package com.liuyi;

 

public class Father {//以后写类都加上public

       private void show() {

              System.out.println("show");

       }

      

       void show2() {

              System.out.println("show2");

       }

      

       protected void show3() {

              System.out.println("show3");

       }

      

       public void show4() {

              System.out.println("show4");

       }

      

       public static void main(String[] args) {

              Father f = new Father();

              f.show();//本类中全部都可以访问

              f.show2();

              f.show3();

              f.show4();

       }

}

局部内部类访问局部变量的注意事项?

              A:局部内部类访问的局部变量必须加final修饰。

              B:为什么呢?

                     因为局部变量使用完毕就消失，而堆内存的数据并不会立即消失。

                     所以，堆内存还是要用该变量，而该变量已经没有了。

                     为了让该值还存在，就加final修饰。

                     通过反编译工具我们看到了，加入final后，堆内存直接存储的是值，而不是                变量名。

       public void method() {

final int num2 = 20;

              class Inner {

                     public void show() {

//从内部类中访问本地变量num2; 本地变量需要被声明为最终类型

                            System.out.println(num2);//20

                     }

              }

Inner i = new Inner();

              i.show();

       }

}

(7)匿名内部类：

A:是局部内部类的简化形式

              B:前提

                     存在一个类或者接口：这里的类可以是具体类也可以是抽象类。

              C:格式:

                     new 类名或者接口名() {

                            重写方法;

                     }

              D:本质：

                     是一个继承了该类或者实现了该接口的子类的匿名对象。

 

interface Inter {

       public abstract void show();

}

class Outer {

       public void method() {

new Inter() {//本质是对象，所以可以new

                     public void show() {//重写接口中的方法

                            System.out.println("show");

                     }

              }.show();//本质是对象，所以可以直接调用方法。这儿只有一个方法需要重写，所                           //以可以直接调用,如果有多个方法需要重写，就要写很多遍这个对象，                          //每一个对象去掉用每一个方法，很麻烦，解决方法见下例

}

class InnerClassDemo6 {

       public static void main(String[] args) {

              Outer o = new Outer();

              o.method();

       }

}

 

 

如果有多个方法需要重写，就应该用多态的方式：

interface Inter {

       public abstract void show();

       public abstract void show2();

}

 

class Outer {

       public void method() {

Inter i = new Inter() { //new Inter()是接口的子类的匿名对象，向上转型为接口：多态

                     public void show() {

                            System.out.println("show");

                     }

                    

                     public void show2() {

                            System.out.println("show2");

                     }

              };

             

              i.show();//用父接口的引用去调用每一个方法，避免了重复写对象的代码

              i.show2();

       }

}

 

class InnerClassDemo6 {

       public static void main(String[] args) {

              Outer o = new Outer();

       }

(8)匿名内部类在开发中的使用

              我们在开发的时候，会看到抽象类，或者接口作为参数。

              而这个时候，我们知道实际需要的是一个子类对象。

       public abstract void study();

}

       //其实这里需要的不是接口，而是该接口的实现类的对象

       }

}

 

//实现类

class Student implements Person {

       public void study() {

              System.out.println("好好学习,天天向上");

       }

}

 

class InnerClassTest2 {
