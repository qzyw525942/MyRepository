把项目打包成jar包并在其他项目中引用

当要用别人的项目中的类时，在自己的类中不能直接使用。这时就要把别人的项目打包成jar包，并弄到自己的项目中再引用。先在要引用的项目上右键---->导出---->选择java---->可运行的jar文件---->下一步---->启动配置处选择要导出的项目，并选择导出的路径和文件名---->点击完成生成jar包。（要导出的项目中应含有具有main方法的类才能选择）

把上面生成的jar包复制到自己的项目根目录下，并在jar包上右键---->构建路径----->添加至构建路径生成引用的库才能使用。

 

一般情况下jar包和说明书配套使用。

参数传递问题

在以下代码的每个方法（包括main方法）的下一行设置断点，调试运行，按F6调试每      一行，看看每一个变量值的变化。

 

//最开始调试界面如下：

class ArgsDemo {

       public static void main(String[] args) {

              int a = 10;//此处设置断点，最开始的断点位于此处

              int b = 20;

              System.out.println("a:"+a+",b:"+b); //a:10,b:20

              change(a,b);//到此处会跳转到参数为int型的change方法中

              System.out.println("a:"+a+",b:"+b); //a:10,b:20，参数为int型的change方法执行                完毕后跳转到这里

 

              int[] arr = {1,2,3,4,5};

              change(arr);//跳转到参数为数组的change方法

              System.out.println(arr[1]); //???    4参数为数组的change方法执行完毕后跳转                 到这里

       }

 

//运行到参数为int型的change方法时，调试界面会多出一个方法：在两个方法中分别       查看各自方法中变量值的变化，可见任凭change方法中a和b两个变量值怎样变化，       mian方法中的二者的值不会变。所以基本类型的变量作为参数传递，传递的是变量的    值，而变量的本身的值不会变化

       public static void change(int a,int b) { //a=10,b=20

              System.out.println("a:"+a+",b:"+b); //a:10,b:20//此处设置断点

              a = b;     //a=20

              b = a + b; //b=40

              System.out.println("a:"+a+",b:"+b); //a:20,b:40

       }//参数为int型的change方法执行完毕后跳回main方法

 

//运行到参数为数组型的change方法时，调试界面会多出一个方法：在两个方法中分别查看各自方法中变量值的变化，可见一旦change方法中数组的值变化，mian方法中的数组的值就会变。所以引用类型的变量作为参数传递，传递的就是变量本身

  

 

public static void change(int[] arr) { //arr={1,2,3,4,5};

              for(int x=0; x<arr.length; x++) {//此处设置断点

                     if(arr[x]%2==0) {

                            arr[x]*=2;

                     }

              }//参数为数组型的change方法执行完毕后跳回main方法

              //arr={1,4,3,8,5};

       }

}

 

第二集   常用类----- Object类

 

Object:类Object是类层次结构的根类。每个类都使用 Object 作为超类。 每个类都直接或者间接的继承自Object类。

 

Object类的方法：

       public Object()：只有一个无参构造方法。所以所有的子类在创建对象时都只能访      

       问父类的无参构造方法

 

             public int hashCode():返回该对象的哈希码值。

                    注意：哈希值是根据哈希算法计算出来的一个值，这个值和对象的地址值有关，                     但是不是实际地址值。你可以理解为地址值。不同的对象对应的哈希值不一样。

public boolean equals(Object obj):指示其他某个对象是否与此对象“相等”。这个方法默认情况下比较的是地址值，但是比较地址值意义不大，所以要重写它，用来比较对象的成员变量值是否相同。Alt+shift+s加上H可以快速生成这个方法

==:

             基本类型：比较的就是值是否相同

            引用类型：比较的就是地址值是否相同

equals:

             引用类型：默认情况下，比较的是地址值。

             但一般情况下都要重写，比较对象的成员变量值是否相同

 

package cn.itcast_03;

 

public class Student

{

       private String name;

       private int age;

 

       public Student()

       {

              super();

       }

 

       public Student(String name, int age)

       {

              super();

              this.name = name;

              this.age = age;

       }

 

       public String getName()

       {

              return name;

       }

 

       public void setName(String name)

       {

              this.name = name;

       }

 

       public int getAge()

       {

              return age;

       }

 

       public void setAge(int age)

       {

              this.age = age;

       }

 

       @Override

       public boolean equals(Object obj)

       {

              if (this == obj)// 为了提高效率，先做基本类型的判断

                     return true;

              if (obj == null)

                     return false;

              if (getClass() != obj.getClass())// 判断obj是不是本类Student的对象，如果不是就返回false,

                     return false;

              Student other = (Student) obj;// 如果是就把obj向下转型，因为父类对象是不能访问子类的成员变量的

if (age != other.age)// 最后的这几句相当于判断：

//this.name.equals(s.name) && this.age == s.age;

                     return false;

              if (name == null)

              {

                     if (other.name != null)

                            return false;

              } else if (!name.equals(other.name))

                     return false;

              return true;// 去除以上为false的情况后就返回true

       }

}

       }

              this.name = name;

       }
