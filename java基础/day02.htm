关键字.

被java语言赋予特定含义的单词就叫做关键字。组成关键字的字母全部小写，String，System和main不是关键字.

在NotePad++等高级记事本中关键字会变色.

标识符.

标识符即给类，变量，方法和接口命名的规则。由大小写字母，数字，$和_组成，不能以数字开头，不能是java中的关键字，区分大小写。

命名原则：见名知意，用有意义的单词命名.      

包名：全部小写：wcw或者cn.itcast.wcw.

常量：全部大写，多个单词间用_隔开：STUDENT或者STUDENT_MAX_AGE.    

接口和类：首字母都大写：Student或者StudentAge.

变量和方法：一个单词的首字母小写，多个单词的驼峰命名法：school或者schoolStudent.

注释.

java代码中的所有注释都会被编译器忽略.

单行注释：//

多行注释：/*   */

文档注释：/**    */    

以后编程中先写注释，说明代码的功能等，然后再写代码，养成好习惯.

常量.

在整个程序执行过程中其值不能发生改变的量.

字符串常量：用双引号引起来的内容：“HelloWorld”  “你好”   内部由几个字符组成.

字符常量：‘A’  ‘a’   ‘2’    内部只能有一个字符,如果有多个字符是字符串常量.

数常量：1213    13.312

布尔常量：true和false.

空常量：null.

它们都可以通过控制台打印出来：

System.out.println(”你好”);//打印出你好.

System.out.println(‘A’);//打印出A.

System.out.println(123);//打印出123.

System.out.println(true);//打印出true.

进制.

x进制就是某一位的数字满x以后就向前进1：逢x进1.

计算机中所有的数据都是转化成二进制来保存和运算的，数据最终转换成bit，每一个bit对应着一个开关，开关的开和关两种状态分别对应着0和1。

数据单位之间的换算：8bit=1byte.

十进制（Decimal system）：默认，无前缀.

二进制（Binary system）：以0b开头.

八进制（Octal number system）：以0开头.

十六进制（Hexadecimal）：以0x开头，后面的a~f大小写皆可以.

0x3c

       =60.

原码，反码和补码.

有符号数据的表示法：数据统一转化成二进制数据后，再用它的原码，反码和补码来表示.

计算机在运算时都是采用补码来计算的.

上面是采用了一个字节的位数来表示，前面也可以不用补0，如+7的原码也可以为0111。

+7和-7在内存中的真实存储就是补码那样的形式.

正数的原码反码和补码都是一样的，注意补码在反码的基础上加1后有可能进位把符号位取为相反.

上图中的最后两题如下：

A:已知某数X的原码为10110100B，试求X的补码和反码。B说明这是二进制数据。如果最前面是0，说明这是正数，无需计算，反码和补码与原码都一样的.         

                     符号位         数值位

             

       原码:     1                   0110100

      

       反码：  1                   1001011

      

       补码：  1                   1001100

      

       B:已知某数X的补码11101110B，试求其原码。

      

                     符号位         数值位

                    

       补码：  1                   1101110（减1为反码）

      

       反码：  1                   1101101（取反为原码）

      

       原码：  1                   0010010

      

由此可见，负数的原码反码和补码在转换中符号位都是不动的，变的只是数值位：从原码到反码数值位取反，从反码到补码数值位加1.

变量.

在程序中其值可以在一定范围内发生改变的量.

定义格式：数据类型 变量名=初始化值：如

int x=1000;

定义一个变量就相当于在内存中开辟了一块空间，为这个空间取的名字就叫做变量名，里面能存入的数据类型就是变量的类型，里面放的具体的值就是变量的值。没有赋值的变量不能直接使用.

右边的数默认都是int型的，前三个数又在byte，short和int的范围内，所以

可以不用加上L，但是加上也不会出错，所以以后凡是long类型的数据都要    加上L.

所以如果上面不加上F,则右边的数据是double型的，double为8个字节，而

                     默认转换(从小范围的转到大范围的).

                  A:byte,short,char—>int—>long—>float—>double：

如果运算中有byte,short,char要先转换为int再参与运算.

                  B:byte,short,char相互之间不转换，他们参与运算首先转换为int类型,其中char                     在转化成int时参照的是ASCII表。

C:boolean类型不参与转换.

如果一个运算中有小范围的数据类型和大范围的数据类型，则小范围的数据类    型默认转化为大范围的数据类型再进行运算，运算的结果是大范围的数据类   型.转换的原则按照上面的原则。

byte a=3;

int  b=4;

//byte c=a+b;//只能从小的范围转到大的范围，运算的结果是大的范围.

int c=a+b//这个才是正确的,1个字节的byte类型转换为4个字节的int类型再       运算.

System.out.println(c);

十进制的3转换为二进制：0011

十进制的4转换为二进制：0100

强制类型转换：从大范围的数据到小范围的数据的转换，格式：

目标数据类型  变量名=（目标数据类型）被转换的数据.

如：byte c=（byte）（a+b）；int型的结果转换为byte型.

面试题2：

byte b = 130;有没有问题?如果我想让赋值正确，可以怎么做?结果是多少呢?

        //因为byte的范围是：-128到127。

              //而130不在此范围内，所以报错。

              //byte b = 130;         

              //我们可以使用强制类型转换

              byte b = (byte) 130;

              //强制类型转换的结果是多少呢?

              System.out.println(b);//结果为-126，怎么来的呢？如下：

       分析过程：

计算的过程是采用补码进行的，所以应该先得到130的补码

              A:获取130这个数据的二进制，130默认为int型，有4个字节：

                     00000000 00000000 00000000 10000010

                     这是130的原码，也是反码，还是补码。

B:对int型的130的补码做截取操作，截成byte类型的了，但是这个结果还是补码。

                     10000010   

              C:已知补码求原码。

                                   符号位         数值位

                     补码：  1                   0000010

                    

                     反码：  1                   0000001

                    

                     原码：  1                   1111110（-126）

所以结果为-126.

过程如下：int型130--->int型原码--->int型反码--->int型补码--->int  型截取-----byte型补码--->byte型反码--->byte型原码--->-126.

 

ASCII码表中只需要记住’a’，’A’，’0’和对应的数字，其他的在他们之上依次增加就可以了：’0’对应的是48.,’A’对应的是65，’a’对应的是97

System.out.println(‘a’);//打印出a

System.out.println(‘a’+1);//打印出98,因为’a’在ASCII码表中对应的是97

System.out.println("hello"+'a'+1); //helloa1，这里的+表示的是字符串连接符，不是加法运算。

System.out.println('a'+1+"hello"); //98hello，前一个+表示加法运算，后一个表示字符串连接符。这两题主要注意要从左到右计算

System.out.println("5+5="+5+5); //5+5=55，同上

System.out.println(5+5+"=5+5"); //10=5+5
