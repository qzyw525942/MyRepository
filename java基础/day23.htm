第一集  概述

 

 

1:引入：

要想了解多线程，必须先了解线程，而要想了解线程，必须先了解进程，因为线程是依赖于进程而存在。

 

2:什么是进程?

         通过任务管理器就能看到进程，可见进程就是正在运行的程序。

         进程是系统进行资源分配和调用的独立单位。每一个进程都有它自己的内存空间和系统         资源。

        

3:多进程的意义?

         单进程的计算机只能同时运行一个程序，而现在的计算机都可以同时运行多个程序。

         比如：一边玩游戏(游戏进程)，一边听音乐(音乐进程)。

         也就是说计算机都是支持多进程的，可以在一个时间段内执行多个任务。这样就提高     CPU 了的使用率。

        

         问题：

                  一边玩游戏，一边听音乐是同时进行的吗?

                  不是。因为一个CPU在某一个时间点上只能做一件事情。

                  而在玩游戏，听音乐的时候，CPU在做着程序间的高效切换，让我们觉得是同                            时进行的。

                 

4:什么是线程?

         同一进程中可同时执行多个任务，每一个任务即为一个线程。

比如：音乐播放器可以一边播放音乐一边显示歌词，这两个任务同时进行，这就是同一         进程(音乐播放器)中的两个线程(边播放音乐边显示歌词)。

 

         线程又叫程序的执行路径

         单线程：如果程序只有一条执行路径。

         多线程：如果程序有多条执行路径。

        

5:多线程的意义?

         多线程是为了提高程序的使用率。

 

         程序的执行其实是在抢CPU的资源即CPU的执行权。

         多个进程在抢这个资源时，某个进程中线程越多，就越能抢到CPU的执行权。

        

 

6：总结：

                  进程：正在运行的程序

                  线程：进程的执行路径

                  单线程：程序只有一条执行路径

                  多线程：程序有多条执行路径

                 

                  多进程的意义?

                           提高CPU的使用率

                  多线程的意义?

                           提高应用程序的使用率

 

 

注意两个词汇的区别：并行和并发。

                  前者指在某一个时间段内同时运行多个程序。

                 后者指在某一个时间点上同时运行多个程序。

 

7:Java程序的运行过程：

                由java命令启动JVM，JVM启动就相当于启动了一个进程。

                 接着由该进程创建了一个主线程去调用main方法。

    

 

                jvm虚拟机启动时除了主线程外，垃圾回收线程也要启动。所以jvm虚拟机的启                动是多线程的。

第四集  线程调度和优先级

 

         A:线程的调度

                  a:分时调度:所有线程平均分配CPU的时间片,轮流使用CPU

                  b:抢占式调度(Java采用此):优先让优先级高的线程使用CPU,它获取的CPU时间片相                  对多一些。如果线程的优先级相同,那么会随机选择一个.

 

         B:获取和设置线程优先级

        

//线程类：

         @Override

         public void run() {

                  for (int x = 0; x < 100; x++) {

                           System.out.println(getName() + ":" + x);

                  }

         }

}

//线程类：

public class ThreadStop extends Thread {

                           ts.interrupt();//使用这个方法，则线程测试类中的线程会被终止，

//但是后面的异常"线程被终止了"和"结束执行”还能被执行

                  } catch (InterruptedException e) {

                           e.printStackTrace();

                  }

         }

}

第九集  线程安全问题及解决方法

 

(1)程序会出现线程安全问题的前提：

                  A:有多个线程

                  B:有共享数据

                  C:有多条操作共享数据的代码

 

这三条是判断一个程序是否有线程安全问题的依据。可见上面继承Thread的线程和实现Runnable的线程都存在这三条出现的问题。A和B两条对应的问题我们改变不了，因为程序必须这样写，所以只能想办法去把C改变一下。

 

怎么改变？

线程安全问题的原因是多个线程同时执行了多条操作共享数据的代码。比如线程1正在执行这多条语句的时候，线程2也在执行这多条语句，所以出现了混乱。如果把多条操作共享数据的代码给包成一个整体，让某个线程在执行的时候，其它线程不能执行，这就能避免出现线程安全问题。这就是下面的同步问题。

 

(2)同步解决线程安全问题  

                  A:同步代码块

                           synchronized(对象) {

                                    需要被同步的代码;

                           }

                          

                           A:对象是什么呢?

                         我们可以随便创建一个对象。

                    B:需要被同步的代码是哪些呢?

                         即多条操作共享数据的代码

                          

注意：

                          同步可以解决线程安全问题的根本原因就在那个对象上。该对象如同锁的功                       能。多个线程必须是同一把锁。

 

//线程类：

public class SellTicket implements Runnable {

         // 定义100张票

         private static int tickets = 100;

         //创建锁对象

         private Object obj = new Object();//定义任意一个对象作为锁

        

         @Override

         public void run() {

 

                  while (true) {

 

                           synchronized (obj) {

//注意这里不能直接把new Object()作为对象传进来，因为这样    //的话每次都会创建对象，每次对应的锁都不一样了，

                                                                                                                 

                                    if (tickets > 0) {

                                             try {

                                                      e.printStackTrace();

                                             }

 

                                             System.out.println(Thread.currentThread().getName()

                                                               + "正在出售第" + (tickets--) + "张票");

                  }

         }

}

public class SellTicket implements Runnable

{

 

         // 定义100张票

         private int tickets = 100;

 

         // 定义同一把锁

         private Object obj = new Object();

 

         @Override

         public void run()

         {

                  while (true)

                  {

                           // t1,t2,t3三个线程都能走到这里,相当于火车上三个人走到了厕                             //所边

//t1抢先一步，先进了厕所的门，然后把门锁上了，t2和t3就    //不能进去了，只能等着

                           synchronized (obj)//锁

                           {

                                    if (tickets > 0)

                                    {

                                             try

                                             {

                                                      Thread.sleep(100); // t1休息一会                                                                                  } catch (InterruptedException e)

                                             {

                                                      e.printStackTrace();

                                             }

 

                                             System.out.println(Thread.currentThread().getName() + "                                                正在出售第" + (tickets--) + "张票 ");

                                             // 窗口1正在出售第100张票

                                    }

                           } // t1完事就出来了，然后就开锁，回到门边。然后t1,t2,t3再                               //次抢机会，看谁抢先一步能进厕所,能抢到的进厕所锁门，不能                                   //抢到的继续等

                  }

         }

}

 

所以同步的前提是：要有多个线程

        注意：多个线程使用的是同一个锁对象

 

 * 同步的好处

 *              同步的出现解决了多线程的安全问题。

 * 同步的弊端

 *              当线程相当多时，因为每个线程都会去判断同步上的锁，这是很耗费         *       资源的，无形中会降低程序的运行效率。
