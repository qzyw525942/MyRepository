第一集  Lock锁

 

         (1)在同步代码块中，不能直接看到在哪里加上了锁，在哪里释放了锁。为了更清晰的表达如何加锁和释放锁，JDK5提供了一个新的接口Lock。

                                        

 * void lock()：上锁。                                                                                        

 * void unlock():释放锁。

 * ReentrantLock是Lock的实现类.//entrant：进入  Reentrant:再入   

 

//线程类：

public class SellTicket implements Runnable {

 

         // 定义票

         private int tickets = 100;

 

         //定义锁对象

         private Lock lock = new ReentrantLock();//多态

 

         @Override

         public void run() {

                  while (true) {

 

                           try {

//为使释放锁的操作一定执行，所以把上锁和放锁的操作放在try语句中

                                    //这样就算中间的代码出了问题，释放锁的代码也一定能执行

                                    lock.lock();//上锁

 

                                    if (tickets > 0) {

                                             try {

                                                      Thread.sleep(100);

                                             } catch (InterruptedException e) {

                                                      e.printStackTrace();

                                             }

                                             System.out.println(Thread.currentThread().getName()

                                                               + "正在出售第" + (tickets--) + "张票");

                                    }

 

                           } finally {

                                    lock.unlock();//释放锁

                           }

                  }

         }

}

第二集  死锁问题

 

同步的弊端：

 *            A:效率低

 *            B:容易产生死锁

 *

 * 死锁：

 *            即多线程在争夺资源的过程中，发生的一种相互等待的现象。

 *

 * 举例：

 *            中国人，美国人吃饭案例。

 *            正常情况：

 *                     中国人:筷子两支

 *                     美国人:刀和叉各一把

 *            现在：

 *                     中国人：筷子1支，刀一把

 *                     美国人：筷子1支，叉一把

 *              情况就是：

 *                       中国人等美国人给筷子，美国人等中国人给刀子，二者互相等待。

                           Object类中提供了三个方法：

                          wait():等待

                         notify():唤醒单个线程

                          notifyAll():唤醒所有线程

 

                         为什么这些方法不定义在Thread类中?

                         这些方法的调用是通过锁对象调用的，而锁对象是任意的对象。

public class SetThread implements Runnable {

         public SetThread(Student s) {

         public void run() {

                  while (true) {

                           synchronized (s) {

                                             } catch (InterruptedException e) {

                                             }

                                    }

//Student类：

public class Student {

         private String name;//这里加上了private

         private int age;

         private boolean flag;

 

         public synchronized void set(String name, int age) {

//同步方法：把同步代码块中的内容放在方法中，并在方法声明上加上一个关键字          //synchronized。此时同步的锁是this。

         //如果资源中有数据，生产者就等待

                  if (this.flag) {//本类的对象调用。开始为false

                           try {

                                    this.wait();//同步的锁是本类的对象this

                           } catch (InterruptedException e) {

                                    e.printStackTrace();

                           }

                  }

 

                  //如果资源中没有数据就设置数据

                  this.name = name;

                  this.age = age;

 

                  //数据生产完修改标记

                  this.flag = true;

                  this.notify();//通知消费者消费数据

         }

 

         public synchronized void get() {

                  // 如果没有数据，消费者就等待

                  if (!this.flag) {//flag在生产者中已经设置为true，所以这里取反

                           try {

                                    this.wait();

                           } catch (InterruptedException e) {

                                    e.printStackTrace();

                           }

                  }

 

                  //如果有数据消费者就获取数据

                  System.out.println(this.name + "---" + this.age);

 

                  //获取完后修改标记

                  this.flag = false;

                  this.notify();//通知生产者生产数据

         }

}

第六集    线程池

 

一般情况下系统创建线程的代价很大。线程池就能避免重复创建线程。因为线程池里的每一个线程代码结束后，线程并不会销毁，而是再次回到线程池中成为空闲状态，等待下一个对象来使用。

 

如何实现线程池的代码?

                A:创建一个线程池对象，参数表示线程池中要创建几个线程对象。

                         public static ExecutorService newFixedThreadPool(int nThreads)//Executor执行者

                 B:这种线程池的线程可以执行Runnable对象或者Callable对象(稍后讲)代表的线程

                         做一个类实现Runnable接口。

                C:调用如下方法即可

                         Future<?> submit(Runnable task)//Future未来，稍后讲

                          <T> Future<T> submit(Callable<T> task)

                 D:结束线程池

                         

//线程类：

public class MyRunnable implements Runnable {

 

         @Override

         public void run() {

                  for (int x = 0; x < 100; x++) {

                           System.out.println(Thread.currentThread().getName() + ":" + x);

                  }

         }

}

 

//线程测试类：

public class ExecutorsDemo {

         public static void main(String[] args) {

                                    //创建一个线程池对象，参数表示线程池中要创建几个线程对象。

                  // public static ExecutorService newFixedThreadPool(int nThreads)

                  ExecutorService pool = Executors.newFixedThreadPool(2);

 

                  // 可以执行Runnable对象或者Callable对象代表的线程

                  pool.submit(new MyRunnable());//传入两个Runnable对象，因为线程池中                      //指定了要创建两个线程

                  pool.submit(new MyRunnable());

 

                  //结束线程池

                  pool.shutdown();

         }

}

 

第七集    实现接口Callable的多线程实现

 

这种方案必须依赖于线程池存在，不能直接使用，所以要通过线程池来体现。

 

//线程类

//Callable:是带泛型的接口,用法和Runnable差不多。它的泛型其实是call()方法的返回值类型。

public class MyCallable implements Callable//先不用泛型

{

                                       

         @Override

         public Object call() throws Exception//实现的接口没使用泛型，所以返回值为object

         {

                  for (int x = 0; x < 100; x++)

                  {

                           System.out.println(Thread.currentThread().getName() + ":" + x);

                  }

                  return null;//未使用泛型，所以返回值设为null

         }

 

}

 

//线程池类：

public class CallableDemo {

         public static void main(String[] args) {

                  //创建线程池对象

                  ExecutorService pool = Executors.newFixedThreadPool(2);

                 

                  //可以执行Runnable对象或者Callable对象代表的线程

                  pool.submit(new MyCallable());

                  pool.submit(new MyCallable());

                 

                  //结束

                  pool.shutdown();

         }

}

第八集    匿名内部类方案的多线程实现

 

匿名内部类的格式：

                 new 类名或者接口名() {

                         重写方法;

                };

                本质：是该类或者接口的子类对象。

 

public class ThreadDemo {

         public static void main(String[] args) {

                  //继承自Thread类来实现多线程

                  new Thread() {  //注意此时run方法在大括号中

                           public void run() {

                                    for (int x = 0; x < 100; x++) {

                                             System.out.println(Thread.currentThread().getName() + ":"

                                                               + x);

                                    }

                           }

                  }.start();

 

                  //实现Runnable接口来实现多线程

                  new Thread(new Runnable() {   //注意此时new Runnable()接口在小括号中

                           @Override

                           public void run() {

                                    for (int x = 0; x < 100; x++) {

                                             System.out.println(Thread.currentThread().getName() + ":"

                                                               + x);

                                    }

                           }

                  }) {                      //大括号在这里，里面没有任何代码

                  }.start();

 

                  //更有难度的

                  new Thread(new Runnable() {//小括号里面有Runnable的run方法

                           @Override

                           public void run() {

                                    for (int x = 0; x < 100; x++) {

                                             System.out.println("hello" + ":" + x);

                                    }

                           }

                  }) {//大括号里面有Thread类的run方法

                           public void run() {

                                    for (int x = 0; x < 100; x++) {

                                             System.out.println("world" + ":" + x);

                                    }

                           }

                  }.start();//此时执行的是大括号里面的run方法，且程序不会报错

         }

}

 

第九集    定时器

 

定时器：可以在指定的时间做某件事情，还可以重复的做某件事情。

依赖Timer和TimerTask这两个类：

 

Timer:定时

                public Timer()

                public void schedule(TimerTask task,long delay)：在指定时间做某事

                public void schedule(TimerTask task,long delay,long period)：每隔一段时间做某事

                public void cancel()：结束定时器

 

                TimerTask:任务

 

案例一：

public class TimerDemo

{

         public static void main(String[] args)

         {

                  // 创建定时器对象

                  Timer t = new Timer();

                  // 3秒后执行爆炸任务

                  // t.schedule(new MyTask(), 3000);

                  // 3秒后执行爆炸任务并结束定时器

                  t.schedule(new MyTask(t), 3000);

         }

}

                  t.cancel();//结束定时器

class DeleteFolder extends TimerTask {//任务类

 

         @Override

         public void run() {

                  File srcFolder = new File("demo");

                  deleteFolder(srcFolder);

         }

 

         // 递归删除目录

         public void deleteFolder(File srcFolder) {

                  File[] fileArray = srcFolder.listFiles();

                           for (File file : fileArray) {

                                    if (file.isDirectory()) {

                                             deleteFolder(file);

                                    } else {

                                             System.out.println(file.getName() + ":" + file.delete());//删除文件

                                    }

                           System.out.println(srcFolder.getName() + ":" + srcFolder.delete());//删除空文件夹

                  }

         }

}

         所以定义在Object里面。

           3:单例设计模式：保证类在内存中只有一个对象。

 

  如何保证类在内存中只有一个对象呢?

                A:把构造方法私有化

                B:在成员位置自己创建一个对象

                C:提供一个公共的方法让外界能提供这个方法访问这个对象。

 

//Student类：

public class Student

{

         //构造私有，不让外界创建对象

         private Student()

         {

         }

         private static Student s = new Student();

         // 为了保证外界能够直接使用该方法，应该加静态

         public static Student getStudent()

                  return s;

         }

}
