第二集   编码表                               

 

              A: 概念

对应的数值组成的一张表。字符是现实生活中的符号，数值是计算              机能认识的符号。编码表把二者结合起来。

             B:常见的编码表

                           ASCII：美国标准信息交换码。用一个字节的后7位表示,最高位是符号位。

                           ISO-8859-1：拉丁码表。欧洲码表。用一个字节的8位表示。

                           GB2312：简体中文编码表。

                           GBK：简体中文编码表升级，融合了更多的中文文字符号。

                           GB18030：GBK的取代版本

Big-5码：繁体中文编码表，俗称大五码

UniCode:国际标准码，融合了多种文字。所有文字都用两个字节来表示,Java      语言使用的就是UniCode

                           UTF-8:最多用三个字节来表示一个字符。即能用一个的就用一个，一个表示不                    了的就用两个，实在不行就用三个

 

 C:字符串中的编码问题

                          

 * String(byte[] bytes, String charsetName):通过指定的编码表解码字节数组

 * byte[] getBytes(String charsetName):使用指定的编码表把字符串编码为字节数组

 *

 * 编码:把看得懂的变成看不懂的

 * String -- byte[]

 *

 * 解码:把看不懂的变成看得懂的

 * byte[] -- String

 *

 * 编码问题核心：只要编码解码的码表是同一个就可以。

 */

面试题：close()和flush()的区别?

 * A:close()先刷新一次缓冲区，再关闭流对象。即close也可以刷新。一般情况下只使用close即可。关闭之后，流对象不可以继续再使用了。

 * B:flush()仅仅刷新缓冲区,刷新之后，流对象还可以继续使用。如果读写的文件比较大才需要flush

 

InputStreamReader的方法：

 * int read():一次读取一个字符

IO流小结

 

第五集   IO流练习

 

字节流复制数据(文本，图片，视频等)共4种方式：

FileOutputStream和FileInputStream一次读取一个字节、一次读取一个字节数组两种;

BufferedOutputStream和BufferedInputStream一次读取一个字节、一次读取一个字节数组两种;

 

字符流复制文本文件共5种方式：

OutputStreamWriter和InputStreamReader一次读取一个字符、一次读取一个字符数组两种；其中FileWriter和FileReader一次读取一个字符、一次读取一个字符数组的也属于这类；

BufferedWriter和BufferedReader一次读取一个字符、一次读取一个字符数组两种；

BufferedWriter和BufferedReader的newLine方法和readLine方法一种

 

案例一:复制文本文件

 

复制数据时，如果数据是用记事本打开能读懂的文本文件，就用字符流，否则用字节流。所以这里应该采用字符流。而字符流有5种方式，所以做这个题目有5种方式。

 

 * 数据源：

 *            c:\\a.txt -- FileReader -- BufferdReader

 * 目的地：

 *            d:\\b.txt -- FileWriter -- BufferedWriter

 */

public class CopyFileDemo

{

         public static void main(String[] args) throws IOException

         {

                  String srcString = "c:\\a.txt";

                  String destString = "d:\\b.txt";

                  // method1(srcString, destString);

                  // method2(srcString, destString);

                  // method3(srcString, destString);

                  // method4(srcString, destString);

                  method5(srcString, destString);

         }

 

         // 第五种方式：高效字符流一次读写一个字符串

         private static void method5(String srcString, String destString) throws IOException

         {

                  BufferedReader br = new BufferedReader(new FileReader(srcString));

                  BufferedWriter bw = new BufferedWriter(new FileWriter(destString));

 

                  String line = null;

                  while ((line = br.readLine()) != null)

                  {

                           bw.write(line);

                           bw.newLine();

                           bw.flush();//三句连写

 *            |--LineNumberReader

 *                     public int getLineNumber()：获得当前行号。

 *                     public void setLineNumber(int lineNumber)：设置当前行号

 

public class LineNumberReaderDemo {

         public static void main(String[] args) throws IOException {

                  LineNumberReader lnr = new LineNumberReader(new FileReader("my.txt"));

 

                  // 从10开始才比较好

                  // lnr.setLineNumber(10);

 

                  // System.out.println(lnr.getLineNumber());

                  // System.out.println(lnr.getLineNumber());

                  // System.out.println(lnr.getLineNumber());

 

                  String line = null;

                  while ((line = lnr.readLine()) != null) {

                           System.out.println(lnr.getLineNumber() + ":" + line);

                  }

 

                  lnr.close();

         }

}
