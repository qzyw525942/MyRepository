第二集   Collection集合

 

2:集合(Collection)

  (1)集合的由来

                  要想存储多个对象，就不能用一个基本的变量来存储，而应该是一个容器类型的变量，在我们目前所学过的知识里面，只有数组和StringBuffer。但是StringBuffer的结果是一个字符串，不一定满足我们的要求，所以我们只能选择数组，这就是对象数组。而对象数组

的长度是固定的，为了适应变化的需求，Java就提供了集合类供我们使用。

        

(2)集合和数组的区别?

                  A:长度区别

                           数组固定

                           集合可变

                  B:内容区别

                           数组可以是基本类型，也可以是引用类型

                           集合只能是引用类型

                  C:元素内容

                           数组只能存储同一种类型

                           集合可以存储不同类型(其实集合一般存储的也是同一种类型)

 

      (3)集合的继承体系结构

                  集合是存储多个元素的，但是存储多个元素也是有不同需求的：比如这多个元素中不能有相同的元素，再比如说要这多个元素按照某种规则排序一下。针对不同的需求，Java就提供了不同的集合类，所以java有很多个集合类。这些集合类的数据存储结构不同。同时这些集合类也有共同点，比如说判断，获取等。把这些集合类的共性内容不断的向上提取，最终就能形成集合的继承体系结构。集合的继承体系结构如下：

/*

 * 集合的遍历。其实就是依次获取集合中的每一个元素。

public class CollectionDemo3 {

         public static void main(String[] args) {

                  Collection c = new ArrayList();

 

                  // 添加元素

                  c.add("hello"); // Object obj = "hello"; 向上转型

                  c.add("world");

                  c.add("java");

 

                  // 遍历

                  // Object[] toArray():把集合转成数组，可以实现对集合的遍历

                  Object[] objs = c.toArray();

                  for (int x = 0; x < objs.length; x++) {

                           // System.out.println(objs[x]);

                           // 元素是字符串，在获取到元素的的同时，如果还想知道元素的长度：

                           // System.out.println(objs[x] + "---" + objs[x].length());

                           // 上面的实现不了，原因是Object中没有length()方法

                           // 我们要想使用字符串的方法，就必须把元素还原成字符串

                           // 所以向下转型

                           String s = (String) objs[x];

                           System.out.println(s + "---" + s.length());

/*

 *            Iterator iterator():迭代器，集合的专用遍历方式

 *            Object next():获取元素,并移动到下一个位置。

 *            boolean hasNext():如果仍有元素可以迭代，则返回 true。

 *            NoSuchElementException：没有这样的元素，因为你已经超出最后一个元素了。

 */

public class IteratorDemo

{

         public static void main(String[] args)

         {

                  // 创建集合对象

                  Collection c = new ArrayList();

 

                  // 创建并添加元素

                  c.add("hello");

                  c.add("world");

                  c.add("java");

 

                  // Iterator iterator():迭代器，集合的专用遍历方式

                  Iterator it = c.iterator(); // 实际返回的肯定是子类对象，因为Iterator 是接口

 

                  while (it.hasNext())

                  {

                           // System.out.println(it.next());

                           String s = (String) it.next();

                           System.out.println(s);

                  }

         }

}

                  C:迭代器的原理和源码。

                           a:迭代器为什么定义为接口而不是类?

如果迭代器定义为一个具体的实现类，那么就可以创建该类的对象，调用该类的方法来实现集合的遍历。但是java提供了很多的集合类，这些集合类的数据结构是不同的，所以他们的存储方式和遍历方式应该是不同的，其中在遍历时，集合应该提供判断功能和获取功能，但是每种集合的这两种功能又不太一样，所以把这两个功能提取出来，这种方式就是接口。所以迭代器就不能定义为具体的实现类，而是一个接口。

                  特点：有序(指存储顺序和取出顺序一致,类似队列)，它的元素可重复。

 

public class ListDemo2 {

         public static void main(String[] args) {

                  // 创建集合对象

                  List list = new ArrayList();//注意所有的集合类都在java.util包中

 

                  // 存储元素

                  list.add("hello");

                  list.add("world");

                  list.add("java");

                  list.add("javaee");

                  list.add("android");

                  list.add("javaee");

                  list.add("android");

 

                  // 遍历集合

                  Iterator it = list.iterator();

                  while (it.hasNext()) {

                           String s = (String) it.next();

                           System.out.println(s);//打印的先后顺序和添加的先后顺序一样，并且内部                             元素可以重复

                  }

         }

}

         (5)并发修改异常(ConcurrentModificationException)

 

        ConcurrentModificationException:当方法检测到对象的并发修改，但不允许这种修改时，

        抛出此异常。

 

/*  问题：

 *            我有一个集合，如下，请问，我想判断里面有没有"world"这个元素，如果有，我

 *              就添加一个"javaee"元素，请写代码实现。

 * 

 * 产生的原因：

 *            迭代器是依赖于集合而存在的，此问题在判断存在"world"这个元素后，集合本身             新添加了元素"javaee"，而迭代器却不知道，所以就报错了，这个错叫并发修改异                  常，见代码表现。

 *            其实这个问题描述的是：迭代器遍历元素的时候，集合是不能修改元素的，但是迭 

 *              代器可以修改元素。

 * 如何解决呢?

 *            A:迭代器迭代元素，迭代器修改元素

 *                     元素是跟在刚才迭代的元素后面的。

 *            B:集合遍历元素，集合修改元素(用普通for循环)

 *                     元素在最后添加。

 */

public class ListIteratorDemo2 {

         public static void main(String[] args) {

                  // 创建List集合对象

                  List list = new ArrayList();

                  // 添加元素

                  list.add("hello");

                  list.add("world");

                  list.add("java");

 

                  // 迭代器遍历

                  // Iterator it = list.iterator();

                  // while (it.hasNext()) {

                  // String s = (String) it.next();

                  // if ("world".equals(s)) {

                  // list.add("javaee");//ConcurrentModificationException，迭代器在遍历集合元素时集             合不能修改元素

                  // }

                  // }

 

//解决方式：

                  // 方式1：迭代器遍历元素，迭代器修改元素

                  // 而Iterator迭代器却没有添加功能，所以我们使用其子接口ListIterator

                  // ListIterator lit = list.listIterator();

                  // while (lit.hasNext()) {

                  // String s = (String) lit.next();

                  // if ("world".equals(s)) {

                  // lit.add("javaee");//迭代器遍历元素迭代器修改元素，不会发生并发异常

                  // }

                  // }

 

                  // 方式2：集合遍历元素，集合修改元素(普通for)

                  for (int x = 0; x < list.size(); x++) {

                           String s = (String) list.get(x);

                           if ("world".equals(s)) {

                                    list.add("javaee");//这种普通for循环也不会发生并发异常

                           }

                  }

 

                  System.out.println("list:" + list);

         }

}

1:集合的由来?

         存储多个对象，可以使用数组，但是数组的长度固定，不适应变化的需求，所以Java就提供了集合。

2:集合和数组的区别?

         数组：长度固定，可以存储基本类型和引用类型，同一种数据类型

         集合：长度可变，只能存储引用类型，可以存储多种引用类型(一般我们仅仅存储一种)

9:并发修改异常出现的原因?解决方案?

         迭代器遍历集合的时候，通过集合去修改了集合的元素。而迭代器是依赖于集合的，它并不知道集合改变了，所以报错。

         B:集合遍历，集合修改集合
