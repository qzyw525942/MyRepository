<style name="AppTheme" parent="AppBaseTheme">//看清楚是在哪个style节点下设置的

第四集  获取版本号

 

上面设置的版本号是在TextView中写死的，但实际开发中应该是动态改变的。这个版本号就在项目的清单文件中：

 

package="com.itheima.mobliesafe75"   //应用在系统中的唯一标示

android:versionCode="1"   //versionCode : 版本号

android:versionName="1.0" >   //versionName : 版本号的名称。上面的版本号实际用的就是这里

 

现在在代码中获取这个版本号并将它设置给TextView显示。

 

在SplashActivity中增加如下方法：

/**

          * 获取当前应用程序的版本号

          *

          * @return

          */

         private String getVersionName()

         {

                  // 包的管理者,用以获取清单文件中的所有信息

                  PackageManager pm = getPackageManager();

                  try

                  {

                           // 根据包名获取清单文件中的信息,其实就是返回一个保存有清单文件信息的javabean

 

                           // packageName :应用程序的包名.可以通过getPackageName()来获取当前应用程序的包名

 

                           // flags:指定信息的标签：

                           // 0:获取基础的信息,比如包名,版本号,要想获取权限等等信息,必须通过标签来指定,才能去获取。

                           // GET_PERMISSIONS标签的含义:处理获取基础信息之外,还会额外获取权限的信息

 

                           PackageInfo packageInfo = pm.getPackageInfo(getPackageName(), 0);

 

                           // 获取版本号名称。packageInfo可以获取清单文件的所有信息

                           String versionName = packageInfo.versionName;

 

                           return versionName;

 

                  } catch (NameNotFoundException e)

                  {

                           // 包名找不到的异常

                           e.printStackTrace();

                  }

                  return null;

         }

 

然后在SplashActivity找到显示版本号的TextView，把上面得到的版本号设置给这个TextView显示：

tv_splash_versionname = (TextView) findViewById(R.id.tv_splash_versionname);

tv_splash_versionname.setText("版本号:" + getVersionName());

 

以后开发的项目会越来越复杂，所以每个控件的id名字不要乱取，否则后面难以辨清。

一般按照如下方式命名：

id :驼峰式命名:控件类型_控件的所在位置_控件表示的逻辑内容，

比如：tv_splash_versionname

 

 

第五集  在splash界面获取软件更新

 

这是一个固定的模式，走的是一整套的流程。步骤如下：

 

A、连接服务器，查看是否有软件的最新版本

 

/**

          * 提醒用户更新版本

          */

         private void update()

         {

                  // 1.连接服务器,查看是否有最新版本.联网操作,属于耗时操作,4.0以后不允许在主线程中执行的,放到子线程中执行

                  new Thread()

                  {

 

                           private int startTime;

 

                           public void run()

                           {

                                    Message message = Message.obtain();

 

                                    // 在连接之前获取一个时间

                                    startTime = (int) System.currentTimeMillis();

 

                                    try

                                    {

                                             // 1.1连接服务器

                                             // 1.1.1设置连接路径

                                             // spec:连接路径

                                             URL url = new URL("http://192.168.20.87:8080/updateinfo.html");

 

                                             // 1.1.2获取连接操作

                                             HttpURLConnection conn = (HttpURLConnection) url.openConnection();// http协议,httpClient

 

                                             // 1.1.3设置超时时间：下面两种方式都可以

                                             conn.setConnectTimeout(5000);// 设置连接超时时间

                                             // conn.setReadTimeout(5000);//设置读取超时时间

 

                                             // 1.1.4设置请求方式

                                             conn.setRequestMethod("GET");// post

 

                                             // 1.1.5获取服务器返回的状态码,200,404,500

                                             int responseCode = conn.getResponseCode();

                                             if (responseCode == 200)

                                             {

                                                      // 连接成功,获取服务器返回的数据

 

                                                      // code:新版本的版本号。将这个版本号和从清单文件获得的版本号进行对比，从而确定是否有新版本

                                                      // apkurl:新版本的下载路径

                                                      // des:描述信息,告诉用户增加了哪些功能,修改那些bug

 

                                                      // 获取数据之前,应该知道服务器是如何封装数据的：xml还是json。这里选择JSon格式

 

                                                      System.out.println("连接成功.....");

 

                                                      // 获取服务器返回的流信息

                                                      InputStream in = conn.getInputStream();

 

                                                      // 将获取到的流信息转化成字符串

                                                      String json = StreamUtil.parserStreamUtil(in);

                                                      // StreamUtil类见后

 

                                                      // 解析json数据

                                                      JSONObject jsonObject = new JSONObject(json);

 

                                                      // 获取数据

                                                      code = jsonObject.getString("code");

                                                      apkurl = jsonObject.getString("apkurl");

                                                      des = jsonObject.getString("des");

 

                                                      System.out.println("code:" + code + "   apkurl:" + apkurl + "   des:" + des);

 

                                                      // 1.2查看是否有最新版本

                                                      // 判断服务器返回的新版本版本号和当前应用程序的版本号是否一致,一致表示没有最新版本,不一致表示有最新版本

                                                      if (code.equals(getVersionName()))

                                                      {

                                                               // 没有最新版本

                                                               message.what = MSG_ENTER_HOME;

                                                      } else

                                                      {

                                                               // 有最新版本

                                                               // 2.弹出对话框,提醒用户更新版本

                                                               message.what = MSG_UPDATE_DIALOG;

                                                      }

                                             } else

                                             {

                                                      // 连接失败

                                                      System.out.println("连接失败.....");

 

                                                      message.what = MSG_SERVER_ERROR;

                                             }

                                    } catch (MalformedURLException e)  //实际开发中最好这样分开抛异常，                                //不要用总的异常，否则分不清到底是哪儿出了问题，后期不好维护

                                    {

                                             e.printStackTrace();

                                             message.what = MSG_URL_ERROR;

                                    } catch (IOException e)

                                    {

                                             e.printStackTrace();

                                             message.what = MSG_IO_ERROR;

                                    } catch (JSONException e)

                                    {

                                             e.printStackTrace();

                                             message.what = MSG_JSON_ERROR;

                                    } finally

                                    {

                                             // 不管有没有异常都会执行这里的代码

 

                                             int endTime = (int) System.currentTimeMillis();

 

                                            

//在这里这样处理的目的，就是为了使用户能看到启动页面上的内      //容，然后过两秒后再弹出对话框，不至于刚启动splash页面，就弹         //出对话框，这样用户就看不到页面上的内容了。

 

// 比较两个时间的时间差,如果小于两秒,睡两秒,大于两秒,不睡

                                             int dTime = endTime - startTime;

                                             if (dTime < 2000)

                                             {

                                                      // 睡两秒钟

                                                      SystemClock.sleep(2000 - dTime);// 始终都是睡两秒钟的时间

                                             }

 

//发送消息的操作放在finally中，无论是否有异常都会执行

                                             handler.sendMessage(message);

                                    }

                           };

                  }.start();

         }

 

Handler如下：

private Handler handler = new Handler()

         {

                  public void handleMessage(android.os.Message msg)

                  {

                           switch (msg.what)

                           {

                           case MSG_UPDATE_DIALOG:

                                    // 有新软件：弹出对话框.方法见后

                                    showdialog();

                                    break;

 

                           case MSG_ENTER_HOME:

                                    // 没有新软件：进入主页，方法见后

                                    enterHome();

                                    break;

 

                           case MSG_SERVER_ERROR:

                                    // 连接失败,服务器出现异常

                                    Toast.makeText(getApplicationContext(), "服务器异常", 0).show();

                                    enterHome();

                                    break;

 

                           case MSG_IO_ERROR:

                                    Toast.makeText(getApplicationContext(), "亲,网络没有连接..", 0).show();

                                    enterHome();

                                    break;

 

                           case MSG_URL_ERROR:

                                    // 方便我们后期定位异常

                                    Toast.makeText(getApplicationContext(), "错误号:" + MSG_URL_ERROR, 0).show();

                                    enterHome();

                                    break;

 

                           case MSG_JSON_ERROR:

                                    Toast.makeText(getApplicationContext(), "错误号:" + MSG_JSON_ERROR, 0).show();

                                    enterHome();

                                    break;

                           }

                  };

         };

第六集    两种上下文的区别

 

1.getApplicationContext()：返回Context

2.activity.this：代表的就是当前的activity,继承自context。根据父类当中有的方法子类中一定有,子类中有的方法父类中不一定有,所以在用getApplicationContext()的时候一定能使用activity.this,但是能用activity.this不一定能使用getApplicationContext()。

 

用错的时候报的异常：android.view.WindowManager$BadTokenException: Unable to add window -- token null is not for an application  没有添加窗口,

 

在使用对话框的时候,必须告诉对话框,要挂载在那个activity中,也就是告诉对话框应用在那个activity中显示，所以必须使用activity.this

 

getContext():也是返回Context ,一般用在单元测试中或者是自定义控件,其实和getApplicationContext()是一样的

查看一个apk文件是否被打包签名过：

将apk后缀名改为.zip并解压，在META-INF中有CERT.RSA文件，用editpad++打开后可隐约看到上面的签名信息的就是被打包过的，如果有个android debug0的就不是打包签名过的。

 

需要注意的是，上面配置生成的.key文件千万不能丢失，密码也不能忘记，发布这个软件新的版本的时候是还需要这个文件和它的密码的，如果丢失文件或者密码是不能发布新版本的。

 

软件新版本发布的两个必要条件：

1、新旧版本的包名必须一致

2、新旧版本的签名必须一致(即.key文件)
