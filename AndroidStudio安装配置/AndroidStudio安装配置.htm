第六集   Material  Design和Android 5.0新特性

 

Material Design是android 5.0提出的设计规范。它对UI中的界面布局以及各个控件的排版，颜色，大小等都提出了限制，是一种UI设计标准。

 

设置项目最小运行版本为Android 5.0及其以上，那么项目中所有使用的布局和控件都将遵循Material Design设计标准。

 

Android 5.0 新特性之一：修改界面标题栏和导航栏颜色，此效果只能在5.0及其以上有效

 

新创建的Module中的MainActivity继承自AppCompatActivity，要修改主题和样式，需要把它修改为继承自Activity；然后在style.xml文件中，作如下修改：

<resources>

 

    <!-- Base application theme. -->

        

//注释掉此行：

    <!--<style name="AppTheme" parent="Theme.AppCompat.Light.DarkActionBar">-->

    //修改为如下：

<style name="AppTheme" parent="android:Theme.Material">

 

//注释掉以下四行：

        <!-- Customize your theme here. -->

        <!--<item name="colorPrimary">@color/colorPrimary</item>

        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>

        <item name="colorAccent">@color/colorAccent</item>-->

                  

                   //增加以下部分，颜色自定义：

 

<!-- 状态栏颜色，会被statusBarColor效果覆盖-->

        <item name="android:colorPrimaryDark">@color/status_red</item>

       

<!-- 状态栏颜色，继承自colorPrimaryDark -->

        <item name="android:statusBarColor">@color/status_red</item>

     

<!-- actionBar颜色 -->

        <item name="android:colorPrimary">@color/action_red</item>

      

<!-- 背景颜色 -->

        <item name="android:windowBackground">@color/window_bg_red</item>

     

<!-- 底部栏颜色 -->

        <item name="android:navigationBarColor">@color/navigation_red</item>

 

    </style>

 

</resources>

 

 

Android 5.0 新特性之二：控件阴影和剪裁效果

 

以前的View只有宽和高，5.0以上提供Z轴参数：

 

android:elevation="5dp"  

//设置该组件“浮”起来的高度，通过设置该属性可以让该组件呈现3D效果。

 

android:translationZ="5dp" 

// 设置该组件在Z方向（垂直屏幕方向）上的位移。

 

剪裁效果如下：

。。。

// ViewOutlineProvider：剪裁类

  ViewOutlineProvider viewOutlineProvider = new ViewOutlineProvider()

        {

            public void getOutline(View view, Outline outline)

            {

                // 可以指定圆形，矩形，圆角矩形；改变不同的方法实现不同的形状

                outline.setOval(0, 0, view.getWidth(), view.getHeight());

                                    

            }

        };

       

   imageview.setOutlineProvider(viewOutlineProvider);

//设定imageview以圆形显示

Android 5.0 新特性之三：图片的颜色

 

提供ring.jpg图片，创建如下tint_bitmap.xml文件：

<bitmap xmlns:android="http://schemas.android.com/apk/res/android"

        android:src="@drawable/ring"

        android:tintMode="multiply"

        android:tint="#5677fc" />

此时tint_bitmap.xml相当于另一张图片了，然后创建如下状态选择器：

<selector xmlns:android="http://schemas.android.com/apk/res/android">

        <item android:drawable="@drawable/tint_bitmap" android:state_pressed="true"/>

        <item android:drawable="@drawable/ring" />

</selector>

把此状态选择器设置给ImageView等控件，就有点击改变图片的效果。

Android 5.0 新特性之五：RecyclerView控件的使用

 

要使用RecyclerView，需要添加RecyclerView依赖：

 

创建项目，然后在界面布局文件中如下：

<android.support.v7.widget.RecyclerView

            android:id="@+id/recycler"

                            android:background="#ff0" //加一个背景色，不然看不到效果

            android:layout_width="match_parent"

            android:layout_height="match_parent"/>

 

// MainActivity如下：

public class MainActivity extends AppCompatActivity

{

    RecyclerView rl;

 

    @Override

    protected void onCreate(Bundle savedInstanceState)

    {

        super.onCreate(savedInstanceState);

        setContentView(R.layout.activity_main);

 

        rl = (RecyclerView) findViewById(R.id.rl);

 

        //设置RecyclerView为垂直排列，效果和ListView一样

//也可以设置为其它方式的排列，比如水平排列等

        LinearLayoutManager manager = new LinearLayoutManager(this);

        manager.setOrientation(LinearLayoutManager.VERTICAL);

        rl.setLayoutManager(manager);

 

        //设置Adapter

        rl.setAdapter(new MyAdapter());

 

    }

 

    class MyAdapter extends RecyclerView.Adapter<ViewHolder>

    {

 

        //创建ViewHolder

        @Override

        public ViewHolder onCreateViewHolder(ViewGroup parent, int viewType)

        {

            //RecyclerView条目布局

            View view = View.inflate(getApplicationContext(), android.R.layout

                    .simple_list_item_1, null);

 

            return new ViewHolder(view);

        }

 

        //为ViewHolder中的控件设置数据

        @Override

        public void onBindViewHolder(ViewHolder holder, int position)

        {

            holder.text1.setText(Cheeses.NAMES[position]);

        }

 

        //返回条目个数

        @Override

        public int getItemCount()

        {

            //Cheeses类中提供NAMES数组，包含很多字符串数据

            return Cheeses.NAMES.length;

        }

    }

 

    static class ViewHolder extends RecyclerView.ViewHolder

    {

 

        TextView text1;

 

        public ViewHolder(View itemView)

        {

            super(itemView);

 

            //构造方法中找到控件

            //text1 = itemView.findViewById(android.R.id.text1);

//？？？老师测试过程中出问题？？？

           

text1 = (TextView) itemView;

        }

 

 

    }

}

 

Android 5.0 新特性之六：SwipeRefreshLayout

 

SwipeRefreshLayout来自v4包，v4包无需添加依赖；这是一个ViewGroup，需要添加子控件ListView，GridView，RecyclerView等；为他们提供下拉刷新的功能

 

界面布局如下：

<android.support.v4.widget.SwipeRefreshLayout 

        android:id="@+id/srl"

        android:layout_width="match_parent"

        android:layout_height="match_parent">

 

        <android.support.v7.widget.RecyclerView

            android:id="@+id/rl"

            android:layout_width="match_parent"

            android:layout_height="match_parent"

            android:background="#ff0"/>

 

    </android.support.v4.widget.SwipeRefreshLayout>

 

代码如下：

。。。

  srl = (SwipeRefreshLayout) findViewById(R.id.srl);

        srl.setOnRefreshListener(new SwipeRefreshLayout.OnRefreshListener()

        {

            //一般在此方法中联网获取新数据，更新界面

            @Override

            public void onRefresh()

            {

                //设置下拉刷新控件3秒以后消失

                srl.postDelayed(new Runnable()

                {

                    @Override

                    public void run()

                    {

                        //是否显示下拉刷新控件

                        srl.setRefreshing(false);

                    }

                }, 3000);

 

            }

        });
