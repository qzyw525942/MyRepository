第一集   IP拨号器案例

 

Android系统内部相当于已经有一个广播电台，它定义了很多广播事件，比如外拨电话、短信到来、sd卡状态 、电池电量变化...android给我们定义了一个组件专门用来接收这些事件，这就是BroadCastReceiver。

 

IP拨号：拨打电话时，如果是长途，就在电话号码之前增加一个短号，否则不加短号

 

界面布局文件如下：

MainActivity如下：

public class MainActivity extends Activity

{

 

         private EditText et_number;

 

         @Override

         protected void onCreate(Bundle savedInstanceState)

         {

                   super.onCreate(savedInstanceState);

                   setContentView(R.layout.activity_main);

 

                   et_number = (EditText) findViewById(R.id.et_number);

         }

 

         // 点击按钮 获取用户输入的iP号码进行保存

         public void click(View v)

         {

                   // [1]获取到用户输入的number

                   String number = et_number.getText().toString().trim();

 

                   // [2]使用sp保存起来

                   SharedPreferences sp = getSharedPreferences("config", 0);

// 第一个参数为xml格式sp文件的名字，第二个参数为文件模式，可以用数字表   //示。这里此方法前面省去了context，其代表当前MainActivity

 

                   // [3]存数据

                   sp.edit().putString("ipnumber", number).commit();

 

                   Toast.makeText(getApplicationContext(), "保存成功", 0).show();

 

         }

 

}

 

//定义一个广播接收者：定义一个类，继承自BroadcastReceiver；复写onReceiver方法

//四大组件都需要在清单文件里面进行配置：与activity同一级：

//<!--配置广播接收者   -->

//<receiver android:name="com.itheima.ipdail.OutGoingCallReceiver">

//    <intent-filter >

//        <action android:name="android.intent.action.NEW_OUTGOING_CALL"/>

//               //这是系统定义的向外拨打电话的事件，一旦手机向外拨打电话，就会触发这个事//                 //件，从而触发BroadcastReceiver的onReceiver方法

//               //这些事件都是系统预先定义的，需要的时候可以查阅资料获得，或者直接在配置//                 //文件中alt+/获得

//    </intent-filter>

//</receiver>

 

 

public class OutGoingCallReceiver extends BroadcastReceiver

{

 

         // 当接收到外拨电话的事件（清单文件中配置的事件）发生的时候，就会执行这个方    //法

         @Override

         public void onReceive(Context context, Intent intent)

         {

 

                   // [0]获取到上面保存的ip号码:这里的Context就表示上面的MainActivity

                   SharedPreferences sp = context.getSharedPreferences("config", 0);

 

                   // [0.1]获取我们保存的ip号码：第二个参数为默认值

                   String ipnumber = sp.getString("ipnumber", "");

 

                   // [1]获取当前我们要拨打的电话号码：清单文件中配置的是向外拨打电话，所以

                   // 下面这个方法获得的数据就是这个外拨电话的电话号码

                   String currentNumber = getResultData();

 

                   // [1.1]判断拨打的电话是否是长途：是否以0开始

                   if (currentNumber.startsWith("0"))

                   {

                            // [2]如果是的话在当前的号码前面加上一个17951

                            setResultData(ipnumber + currentNumber);

                   }

 

         }

 

}

 

增加拨打电话的权限。

<uses-permission android:name="android.permission.PROCESS_OUTGOING_CALLS"/>

 

在MainActivity的界面中输入IP短号并点击保存；打开拨号器，在系统拨号器中拨打电话，，(注意此时上面这个程序的进程还没结束，程序还在后台运行)就会触发清单文件中配置的事件，从而触发onReceiver方法，也就会被上面的BroadcastReceiver收听到，如果号码以0开始，在这个号码之前增加短号。

 

 

第二集   SD卡状态监听案例

 

MainActivity的界面和java文件都是工程的初始化状态，无需改变。

 

receiver的清单配置文件如下：

<receiver android:name="com.itheima.sdcardstate.SdcardReceiver">

    <intent-filter >

      <action android:name="android.intent.action.MEDIA_UNMOUNTED"/>

      <action android:name="android.intent.action.MEDIA_MOUNTED"/>

      <!-- 同一个receiver可以配置多个action，也就是说可以用多个事件去触发receiver-->

       <action android:name="android.intent.action."/>

               

      <!--想让上面的这2个事件生效必须加上如下一个data约束。需要加data约束的事件          很少，这里的两个事件只是两个特例之一，后面还会遇到一个需要加约束的事件-->

        <data android:scheme="file"/>

    </intent-filter>

</receiver>

 

 

BroadcastReceiver如下：

public class SdcardReceiver extends BroadcastReceiver

{

 

         // 当sd卡被卸载或者被挂载上来的时候（清单文件中已配置这两个事件）会执行onReceive方法

         @Override

         public void onReceive(Context context, Intent intent)

         {

 

                   // [1]获取到当前广播的事件类型：清单文件中配置了多个事件，

                   // 先用这个方法获得它们，下面再判断这些事件

                   String action = intent.getAction();

 

                   // [2]对action做一个判断

                   if ("android.intent.action.MEDIA_UNMOUNTED".equals(action))

                   {

                            System.out.println("说明sd卡 卸载了");

 

                   } else if ("android.intent.action.MEDIA_MOUNTED".equals(action))

                   {

 

                            System.out.println("说明sd卡挂载了");

                   }

 

         }

 

}

 

// 测试这个案例要用2.3的模拟器，因为4.X的没有卸载SD卡的选项。设置中有以下两个选项，就是卸载和挂载SD卡的选项，分别选择这两个选项，就会触发清单文件中配置的事件，从而触发receiver，执行onReceiver方法

 

 

第三集   短信窃听器案例

 

MainActivity的界面和java文件都是工程的初始化状态，无需改变。

 

receiver的配置文件如下：

<receiver android:name="com.itheima.smslistener.SmsListenerReceiver">

       <intent-filter >

       <action android:name="android.provider.Telephony.SMS_RECEIVED" />、

//监听短信接收事件，当本机接收到其他人发来的短信时触发此事件

       </intent-filter>

 </receiver>

 

BroadcastReceiver如下：

public class SmsListenerReceiver extends BroadcastReceiver

{

 

         // 当短信到来的时候就会执行这个方法

         @Override

         public void onReceive(Context context, Intent intent)

         {

 

                   // [1]获取所发短信的号码和内容

                   Object[] objects = (Object[]) intent.getExtras().get("pdus");

                  // 一个pdu表示一条短信，多个pud表示多条短信

 

                   for (Object pdu : objects)

                   {

 

                            // [2]获取smsmessage实例

                            SmsMessage smsMessage = SmsMessage.createFromPdu((byte[]) pdu);

 

                            // [3]获取发送短信的内容

                            String body = smsMessage.getMessageBody();

 

                            // [4]获取发送者：Originating原始的

                            String address = smsMessage.getOriginatingAddress();

 

                            System.out.println("body:" + body + "---" + address);

 

                            // [5]比如发送短信扣流量

 

                   }

         }

}

 

增加接收短信的权限;

<uses-permission android:name="android.permission.RECEIVE_SMS"/>

 

这样当别人发短信给本机时就会执行onReceiver方法，从而窃听短信。

 

上面的程序在手机上还有图标存在，比如：

要想隐藏程序的图标，只需要在配置文件中把设置为程序启动后默认加载的第一个activity的：

 <intent-filter>

        <action android:name="android.intent.action.MAIN" />

 

        <category android:name="android.intent.category.LAUNCHER" />

 </intent-filter>

部分去掉，则程序安装后就没有图标存在了，用户也就找不到程序了，也就没法卸载了。

 

但是在4.1之后的系统中，如果一个应用程序没有界面，则里面的广播事件无效。

 

 

第四集  程序的卸载安装监听案例

 

MainActivity的界面和java文件都是工程的初始化状态，无需改变。

 

receiver的配置文件如下：

<receiver android:name="com.itheima.appstate.AppStateReceiver">     

      <intent-filter >

          <action android:name="android.intent.action.PACKAGE_INSTALL"/>

           <action android:name="android.intent.action.PACKAGE_ADDED"/>

           <action android:name="android.intent.action.PACKAGE_REMOVED"/>

               

           <!-- 这里也需要为上面这几个事件配置约束，这是两个特例之二 -->

           <data android:scheme="package"/>

               

    </intent-filter>

 </receiver>

 

BroadcastReceiver如下：

public class AppStateReceiver extends BroadcastReceiver

{

 

         @Override

         public void onReceive(Context context, Intent intent)

         {

 

                   // 获取到当前广播的事件类型

                   String action = intent.getAction();

 

                   if ("android.intent.action.PACKAGE_INSTALL".equals(action))

                   {

 

                            System.out.println("应用被安装了");

 

                   } else if ("android.intent.action.PACKAGE_ADDED".equals(action))

                   {

                            System.out.println("~~~~~~应用被安装了");

 

                   } else if ("android.intent.action.PACKAGE_REMOVED".equals(action))

                   {

 

                            System.out.println("应用被卸载了" + intent.getData());

//获得安装的应用的包名

 

                   }

 

         }

 

}

可见安装应用时触发的是PACKAGE_ADDED事件，PACKAGE_INSTALL是预留的，没有含义

public class BootReceiver extends BroadcastReceiver {

第七集  特殊的广播事件

 

对于操作特别频繁的广播事件，如屏幕的锁屏和解锁、电池电量的变化，这样的广播接收者在清单文件里面注册无效，只能动态注册BroadcastReceiver。

 

MainActivity如下：

public class MainActivity extends Activity

{

 

         private ScreenReceiver screenReceiver;

 

         @Override

         protected void onCreate(Bundle savedInstanceState)

         {

                   super.onCreate(savedInstanceState);

                   setContentView(R.layout.activity_main);

 

                   // 静态注册无效

                   /*

                    * <receiver android:name="com.itheima.screen.ScreenReceiver">

                    * <intent-filter >

                    * <action android:name="android.intent.action.SCREEN_OFF"/>

                    * <action android:name="android.intent.action.SCREEN_ON"/>

                    * </intent-filter>

                    * </receiver>

                    */

 

                   // [1]动态的去注册屏幕解锁和锁屏的广播

                   screenReceiver = new ScreenReceiver();

 

                   // [2]创建intent-filter对象

                   IntentFilter filter = new IntentFilter();

 

                   // [3]添加要注册的action

                   filter.addAction("android.intent.action.SCREEN_OFF");

                   filter.addAction("android.intent.action.SCREEN_ON");

 

                   // [4]注册广播接收者

                   this.registerReceiver(screenReceiver, filter);

 

         }

 

         @Override

         protected void onDestroy()

         {

                   // 当activity销毁的时候 取消注册广播接收者：必须要取消注册，否则退出activity                  //时

                   unregisterReceiver(screenReceiver);

 

                   super.onDestroy();

         }

 

}

 

BroadcastReceiver如下：

public class ScreenReceiver extends BroadcastReceiver

{

 

         @Override

         public void onReceive(Context context, Intent intent)

         {

 

                   // [1]获取到当前广播的事件类型

                   String action = intent.getAction();

 

                   // [2]对当前广播事件类型做一个判断

 

                   if ("android.intent.action.SCREEN_OFF".equals(action))

                   {

 

                            System.out.println("屏幕锁屏了");

                   } else if ("android.intent.action.SCREEN_ON".equals(action))

                   {

 

                            System.out.println("说明屏幕解锁了~~~");

                   }

 

         }

 

}

        <item name="android:textSize">45sp</item>

       

</style>

 

这样定义好后，TextView的语句就可以如下：

<TextView

        style="@style/my_style"

        android:text="ccc" />

 

以后要统一设置几个TextView的样式，就不必一个一个的去修改，而是直接在style.xml文件中统一修改就可以了。

 

B、主题

 

如果要设置界面的背景色，同样可以在style.xml文件中：

<style name="my_theme">

        <item name="android:background">#ff9999</item>

       

    </style>

 

设置好后，在清单文件中的application节点下通过如下方式引用：

 

android:theme="@style/my_theme"

 

 

样式和主题：  

共同点:(1)定义的方式是一样的

不同点:(1)style作用范围主要是各个控件 (button、textview)

 theme作用范围在activity或者Application节点下

 

 

第九集   国际化

 

国际化又叫做i18n。

 

将布局文件中所有的汉字的部分抽取到/values/strings.xml文件中去，比如：

android:text="@string/app_name"

 

在res目录下新建程序需要面向的国家的语言，比如软件需要向台湾人开放，就新建一个

values-zh-rTW目录，把上面的values中的strings.xml复制一份，再把里面的所有汉字变为繁体中文。

 

当手机的语言环境设置为繁体中文后，程序使用的语言也就随着语言环境而使用繁体中文的/values-zh-rTW/strings.xml文件来显示界面语言
