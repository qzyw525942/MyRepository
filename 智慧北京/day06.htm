第一集  图片的三级缓存

 

- 优先从内存中加载图片, 速度最快, 不浪费流量

- 其次从本地SD卡加载图片, 速度快, 不浪费流量

- 最后从网络下载图片, 速度慢, 浪费流量                  

 

以下三级缓存在组图模块中使用。

NetCacheUtils如下：

public class NetCacheUtils

{

 

        // 从网络获得图片

        public void getBitmapFromNet(ImageView imageView, String url)

        {

                new BitmapTask().execute(imageView, url);// 启动AsyncTask

        }

 

        // AsyncTask：异步封装的工具类，可以实现异步网络请求及主界面更新（对线程池+Handler的封装）

        /*

         * 第一个泛型参数的意义：doInBackground方法的参数，也就是execute方法的参数

         * 第二个泛型参数的意义：onProgressUpdate方法的参数 第三个泛型参

         * 数的意义：onPostExecute方法的参数，以及doInBackground方法的返回值类型

         * 这几个参数的意义可以依次修改以后看哪些地方会报错，就能看出来了

         */

        class BitmapTask extends AsyncTask<Object, Integer, Bitmap>

        {

                private ImageView imageView;

                private String url;

 

                // 预加载：运行在主线程，可更新UI

                @Override

                protected void onPreExecute()

                {

                        super.onPreExecute();

 

                }

 

                // 正在加载：运行在子线程

                @Override

                protected Bitmap doInBackground(Object... params)

                {

                        imageView = (ImageView) params[0];

                        url = (String) params[1];

 

                        // 为每个ImageView打一个标记，防止错乱

                        imageView.setTag(url);

 

                        Bitmap bitmap = download(url);

 

                        return bitmap;

                }

 

                // 加载结束：运行在主线程，可更新UI

                @Override

                protected void onPostExecute(Bitmap bitmap)

                {

                        // 这里的bitmap也就是doInBackground返回的值

                        super.onPostExecute(bitmap);

 

                        if (bitmap != null)

                        {

                                String url = (String) imageView.getTag();

 

                                if (url.equals(this.url))

// 判断图片绑定的url是不是就是当前图片bitmap的url，如果//是，就说明图片正确。之所以要这样，是为了防止每个//ImageView在显示图片时错乱。

                                {

                                        imageView.setImageBitmap(bitmap);

 

                                }

 

                        }

                }

 

                // 获得网络加载的进度：运行在主线程，可更新UI

                @Override

                protected void onProgressUpdate(Integer... values)

                {

                        super.onProgressUpdate(values);

                }

        }

 

        // 下载图片

        public Bitmap download(String url)

        {

                HttpURLConnection connection = null;

 

                try

                {

                        connection = (HttpURLConnection) new URL(url).openConnection();

 

                        connection.setRequestMethod("GET");

                        connection.setConnectTimeout(5000);// 连接超时

                        connection.setReadTimeout(5000);// 读取超时

                        connection.connect();// 连接

 

                        if (connection.getResponseCode() == 200)

                        {

                                InputStream is = connection.getInputStream();

 

                                Bitmap bitmap = BitmapFactory.decodeStream(is);

 

                                return bitmap;

 

                        }

 

                } catch (Exception e)

                {

                        e.printStackTrace();

 

                } finally

                {

                        if (connection != null)

                        {

                                connection.disconnect();

                        }

                }

 

                return null;

 

        }

}

 

第三集  本地SD卡缓存

 

public class LocalCacheUtils

{

        public static final String LOCAL_CAHCE_PATH = Environment

                        .getExternalStorageDirectory().getAbsolutePath() + "/ZhiHuiBeiJing";

 

        // 设置缓存

        public void setCache(String url, Bitmap bitmap)

        {

 

                File file = new File(LOCAL_CAHCE_PATH);

                if (!file.exists() || !file.isDirectory())// 如果file不存在或者不是文件夹

                {

                        file.mkdir();// 创建文件夹

                }

 

                try

                {

                        // 每张图片对应着一个url，依此作为图片的文件名。url中可能包含特殊的字符，所以使用MD5加密

                        String bitmapName = MD5Encoder.encode(url);// MD5Encoder网上有

 

                        File cacheFile = new File(file, bitmapName);// 创建文件

 

                        // 图片压缩.第二个参数是压缩比例

                        bitmap.compress(CompressFormat.JPEG, 100, new FileOutputStream(

                                        cacheFile));

 

                } catch (Exception e)

                {

                        e.printStackTrace();

                }

 

        }

 

        // 获得缓存

        public Bitmap getCache(String url)

        {

                Bitmap bitmap = null;

 

           try

                {

                        File cacheFile = new File(LOCAL_CAHCE_PATH, MD5Encoder.encode(url));

 

                        if (cacheFile.exists())

                        {

                                bitmap = BitmapFactory.decodeStream(new FileInputStream(

                                                cacheFile));

 

                        }

 

                        return bitmap;

 

                } catch (Exception e)

                {

                        e.printStackTrace();

                }

 

                return null;

        }

}

第四集  内存缓存

 

代码如下：

public class MemoryCacheUtils

{

        // SoftReference:软引用。用其将Bitmap包装起来，便于垃圾回收器适时回收Bitmap，避免OOM

 

        HashMap<String, SoftReference<Bitmap>> hashMap = new HashMap<String, SoftReference<Bitmap>>();

 

        public void setMemoryCache(String url, Bitmap bitmap)

        {

                SoftReference<Bitmap> reference = new SoftReference<Bitmap>(bitmap);

 

                hashMap.put(url, reference);

        }

 

        public Bitmap getMemoryCache(String url)

        {

                SoftReference<Bitmap> reference = hashMap.get(url);

 

                if (reference != null)

                {

                        Bitmap bitmap = reference.get();

 

                        return bitmap;

 

                }

 

                return null;

        }

}

第五集  屏幕适配

 

- 图片适配

       

        ldpi:240*320 

        mdpi: 320*240

        hdpi: 480*800

        xhdpi: 1280*720

        xxhdpi: 1920*1080

 

        常规做法: 做一套切图放在hdpi或xhdpi下. 如果某个屏幕出了问题, 再针对该屏幕, 对相应出问题的图片进行替换.

 

- 布局适配(不太常用)

 

        创建layout-800x480文件夹:专门存放针对480*800屏幕适配的布局文件, 一般只调整位置和大小, 不建议对控件类型和个数进行调整

 

- 尺寸适配(很常用)

       

        //dp和px的关系：dp = px/设备密度

 

获取设备密度:

        float density = this.getResources().getDisplayMetrics().density;

 

如果在布局文件中设置480*800的某个控件的宽度为25dp；而在240*320上显示的效果不太理想。此时可以新建values-320x240/dimens.xml；

<resources>

 

    <dimen name="width">20dp</dimen>

        //修改为240*320上可以显示完美的值

 

</resources>

 

在设置此控件的宽度为：

  

    <TextView

        android:layout_width="@dimen/width"

        android:layout_height="80dp"

        android:background="#f00" />

 

这样在240*320的设备上也能进行屏幕适配了。

 

- 权重适配：用于线性布局

 

         android:weightSum="3"

android:weight="3"

 

 

- 代码适配

//获取屏幕的宽和高

WindowManager wm = getWindowManager();

int width = wm.getDefaultDisplay().getWidth();

int height = wm.getDefaultDisplay().getHeight();

 

TextView tvText = (TextView) findViewById(R.id.tv_text);

 

//设置tvText的宽和高为屏幕宽和高的1/3和1/5.

LinearLayout.LayoutParams params = (LayoutParams) tvText.getLayoutParams();

params.width = width / 3;

params.height = height / 5;

 

tvText.setLayoutParams(params);

 

这样设置好后,无论在什么大小的屏幕上都是一样的效果了。

 

> 养成良好的开发习惯: 多用dp,sp,不用px; 多用线性布局和相对布局, 不用绝对布局; 代码中如果必须设置像素的话, 将dp转为px进行设置

1、新手引导页底部小圆点屏幕适配问题

在GuideActivity的initData方法中找到如下：

                        {

要想屏幕适配，这里就应该用dp，所以要把px转化为dp。

 

//dp和px的转化工具类

public class DensityUtils

{

        public static int dip2px(Context context, int dp)

        {

                float density = context.getResources().getDisplayMetrics().density;

 

                int px = (int) (dp * density + 0.5f);

                // 加上0.5的原因是：

                // 4.9+0.5--->5.4--->int后为5

                // 4.1+0.5---4.6--->int后为4

                // 否则4.9和4.1 int后为4，差别太大

 

                return px;

        }

 

        public static int px2dip(Context context, int px)

        {

                float density = context.getResources().getDisplayMetrics().density;

 

                int dp = (int) (px / density);

 

                return dp;

        }

}

 

GuideActivity中的代码修改如下：

if (i > 0)

                        {

                                // 从第二个小圆点开始，距离左边小圆点10dp

                                params.leftMargin = DensityUtils.dip2px(this, 10);

                        }

 

这样就能实现屏幕适配。
